### 2.2 关键实现点

#### 实现点一：把登录入口从静默注册切换为 SSO 回调建会话

现在的链路是：

1. 前端带科技平台 token 进入 `GET /api/auth/sso-callback`
2. ABS 服务端调用科技平台用户信息接口校验 token
3. 根据上游用户信息即时建档或复用本地会员
4. 生成本地 JWT 并写入 Cookie
5. 前端通过 `GET /api/users/me` 判断本地登录态

这条链路落地后，业务接口不再依赖注册接口作为前置入口。

#### 实现点二：把业务会员身份从“外部参数”切换为“认证上下文”

现在的业务接口不再信任外部传入的 `memberId`，而是统一从认证上下文解析当前会员。这是今天最重要的安全边界调整之一。

#### 实现点三：把浏览器态和回调态的鉴权方式明确拆开

当前边界是：

1. `/api/equity/**`、`/api/users/me` 使用 JWT + Cookie。
2. `/api/callbacks/**` 保持签名认证。

这样后续无论扩前端能力还是扩外部合作接口，都不会混乱在同一套鉴权体系里。

1. 后续类似改造建议固定按四层设计：入口、会话、业务授权、外部协作。
2. 对复用本地数据库的项目，建议保留真实数据库集成测试，并允许必要的增量 schema 对齐。
