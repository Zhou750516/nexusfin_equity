## 2. 今日结论

1. 在正确的本地联调启动参数下，H5 页面维度主流程已跑通
2. 本轮异常的主要根因是环境配置错误，不是当前业务代码本身失效
3. H5 本地联调后续必须统一使用固定启动命令，不可只切 MySQL、不切 Yunka `REST`
4. MySQL 基线仍需补齐 `HUXUAN_CARD`，否则首页首次借款申请会报 `PRODUCT_NOT_FOUND`
5. `payment_record` 本轮未新增 H5 还款记录，后续需确认该表是否属于当前 H5 还款链路的必需落库范围

## 3. 风险与注意事项

1. 若遗漏 `--nexusfin.third-party.yunka.mode=REST`，页面会出现“假异常”
2. 若未补支付协议，借款 / 权益链路无法稳定推进
3. 若未预置 `HUXUAN_CARD`，H5 主流程无法直接起跑
