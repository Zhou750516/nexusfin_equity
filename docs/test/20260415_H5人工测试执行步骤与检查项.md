```js
(async () => {
  const encoder = new TextEncoder();
  const ts = String(Math.floor(Date.now() / 1000));
  const nonce = 'manual-' + Date.now();
  const secret = 'abs-secret-key';
  const appId = 'abs-app';
  const signText = `${appId}|${ts}|${nonce}`;
  const key = await crypto.subtle.importKey(
    'raw',
    encoder.encode(secret),
    { name: 'HMAC', hash: 'SHA-256' },
    false,
    ['sign']
  );
  const sigBuffer = await crypto.subtle.sign('HMAC', key, encoder.encode(signText));
  const signature = Array.from(new Uint8Array(sigBuffer))
    .map(b => b.toString(16).padStart(2, '0'))
    .join('');

  const requestId = 'manual-protocol-' + Date.now();
  const protocolNo = 'AIP-MANUAL-' + Date.now();

  const resp = await fetch('/api/callbacks/payment-protocol', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-App-Id': appId,
      'X-Timestamp': ts,
      'X-Nonce': nonce,
      'X-Signature': signature
    },
    body: JSON.stringify({
      requestId,
      memberId: null,
      externalUserId: 'tech-user-local-001',
      providerCode: 'ALLINPAY',
      protocolNo,
      protocolStatus: 'ACTIVE',
      signRequestNo: requestId,
      channelCode: 'KJ',
      signedTs: '2026-04-15T16:58:00'
    }),
    credentials: 'include'
  });

  console.log(resp.status, await resp.text());
})();
```

检查项：

- MySQL 中已新增 `ACTIVE` 的支付协议

## 6. 页面人工测试步骤

> 建议在浏览器打开 H5 页面后，同时打开 DevTools 的 `Network`，并筛选 `api`，方便每一步核对接口。

### 步骤 1：首页加载

打开：

```text
http://127.0.0.1:3000/
```

页面要检查什么：

- 借款金额显示 `¥3,000`
- 借款期数显示 `3期`
- 收款账户显示 `招商银行 8648`
- 页面可以正常操作，无明显报错

接口要检查什么：

- `GET /api/loan/calculator-config`
- `POST /api/loan/calculate`

如何检查：

1. 在 Network 中点开 `calculator-config`
   - 确认返回 200
   - 确认返回了金额范围、期数、收款账户
2. 在 Network 中点开 `calculate`
   - 确认返回 200
   - 确认返回了利息 / 还款计划等信息

通过标准：

- 页面配置与试算均正常展示

### 步骤 2：确认借款

操作：

1. 勾选协议
2. 点击“确认借款”

页面要检查什么：

- 页面跳转到 `/approval-pending`

接口要检查什么：

- `POST /api/loan/apply`

如何检查：

在 Network 中点开 `loan/apply`，检查：

- 返回 200
- `code=0`
- `data.applicationId` 非空
- `data.status = pending`
- `data.benefitsActivated = true`

通过标准：

- 页面成功跳转审批中页
- 接口成功返回 `applicationId`

异常提示：

- 如果这里报：`PRODUCT_NOT_FOUND:Benefit product not found`
- 说明 `HUXUAN_CARD` 没准备好，先回到第 5.1 步处理

### 步骤 3：审批中页

页面要检查什么：

- 标题显示“借款审批中”
- 页面显示步骤：
  - 提交申请
  - 审批中
  - 等待放款

接口要检查什么：

- `GET /api/loan/approval-status/{applicationId}`

如何检查：

1. 在 Network 中找到 `approval-status`
2. 确认返回 200
3. 确认首次返回可能是：
   - `status = reviewing`

通过标准：

- 页面能正确显示审批中状态

说明：

- 审批中页第一次看到 `reviewing` 是正常的
- 后续会根据 Yunka stub 的状态推进进入审批结果页

## 8. 最关键的 6 个人工判断点

人工测试时，优先抓下面 6 个点：

1. 后端启动命令里是否带了：
   - `--nexusfin.third-party.yunka.mode=REST`
2. `benefit_product` 中是否有：
   - `HUXUAN_CARD`
3. 当前联调用户是否有：
   - `ACTIVE` 支付协议
4. `POST /api/loan/apply` 返回后：
   - `applicationId` 是否非空
5. `GET /api/loan/approval-result/{applicationId}` 返回时：
   - 是否 `approved`
   - `loanId` 是否非空
6. `GET /api/repayment/info/{loanId}` 返回时：
   - `repaymentAmount` 是否为 `1018.50`

如果这 6 个点都成立，页面主流程通常就能顺利跑通。

## 9. 出现异常时如何快速定位

### 场景 1：首页点“确认借款”直接失败

现象：

- 返回 `PRODUCT_NOT_FOUND`

优先检查：

- `benefit_product` 表中是否存在 `HUXUAN_CARD`

### 场景 2：页面一直审批中，最后又变拒绝

现象：

- `approval-status` 长期 `reviewing`
- `approval-result` 变成 `rejected`

优先检查：

- 后端是不是仍然以 `YUNKA_MODE=MOCK` 启动

### 场景 3：还款页金额变成 `0.00`

现象：

- `repaymentAmount = 0.00`
- 页面显示 `确认支付 ¥0`

优先检查：

- 后端是否漏掉：
  - `--nexusfin.third-party.yunka.mode=REST`
  - `--nexusfin.third-party.yunka.base-url=http://127.0.0.1:18081`

### 场景 4：页面像没登录

现象：

- 接口返回未认证
- 页面一直无法正常取数

优先检查：

- 浏览器中是否存在 `NEXUSFIN_AUTH` Cookie
- 后端是否带了：
  - `--nexusfin.auth.jwt.cookie-secure=false`

### 场景 5：借款 / 权益链路推进不稳定

现象：

- 页面能打开，但开通或借款链路走不顺

优先检查：

- 当前用户是否有 `ACTIVE` 支付协议

## 10. 一句话执行口径

> 人工测试时，先保证“环境启动正确 + 前置数据齐全”，再从首页按主流程一步步核对页面、接口和 MySQL。若出现审批拒绝或还款金额为 0，优先检查后端是否误跑在 `YUNKA_MODE=MOCK`。
