### 第三步：人工走查时看执行步骤

- `test/20260415_H5人工测试执行步骤与检查项.md`

这份文档解决“**我自己怎么一步步手工测、每一步重点看什么、出了问题先查哪里**”的问题，适合实际打开浏览器边测边核对时使用。

压缩总结如下：

- 先启动 H5、Java 后端、科技平台 stub、云卡 stub
- 后端启动必须带：
  - `--nexusfin.third-party.yunka.mode=REST`
  - `--nexusfin.third-party.yunka.base-url=http://127.0.0.1:18081`
- 联调前必须确认：
  - `benefit_product` 中存在 `HUXUAN_CARD`
  - 当前用户已拿到 `NEXUSFIN_AUTH`
  - 当前用户已补入 `ACTIVE` 支付协议
- 页面主流程按以下顺序人工走查：
  - 首页
  - 借款申请
  - 审批中页
  - 权益详情 / 开通
  - 审批结果页
  - 还款确认页
  - 还款成功页
- 每一步同时检查三类信息：
  - 页面展示是否符合预期
  - `Network` 中对应 `/api/*` 接口是否成功
  - MySQL 关键表是否已写入必要数据
- 如果出现以下现象，优先检查：
  - `approval-result` 变成拒绝 → 先查 Yunka 是否误跑在 `MOCK`
  - `repaymentAmount = 0.00` → 先查 Yunka 是否误跑在 `MOCK`
  - `PRODUCT_NOT_FOUND` → 先查 `HUXUAN_CARD`
  - 页面像未登录 → 先查 `NEXUSFIN_AUTH`

## 3. 当前最关键的联调口径

本地 H5 联调时，必须同时满足以下四个条件：

1. 后端已连接真实 MySQL；
2. 后端已切到云卡 `REST` stub，而不是默认 `MOCK`；
3. `benefit_product` 中已存在 `HUXUAN_CARD`；
4. 当前联调用户已补入有效支付协议。

如果缺少其中任意一项，页面链路都可能出现假异常。
