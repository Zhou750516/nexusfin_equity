# 2026-04-29 云卡 / 齐为 Mock 切换说明

## 目标

为本地调试、focused 测试和 H5 全流程联调提供一套可复用的启动时手动切换口径，避免把“应用内 mock”与“外部本地 stub”混为一谈。

## 唯一真值

### Yunka

- `YUNKA_MODE=MOCK`
  - 走应用内 `RestYunkaGatewayClient` mock 分支
  - 不发 HTTP
  - 返回固定成功外壳：`code=0`、`message=MOCK`、`data={}`
  - 适合轻量本地调试、非全流程场景、focused 单测
- `YUNKA_MODE=REST`
  - 走 `RestYunkaGatewayClient` HTTP 分支
  - 请求目标为 `nexusfin.third-party.yunka.base-url + gateway-path`
  - 适合真实联调，或指向本地 `18081` Yunka stub 做 H5 happy-path
- 非法 `YUNKA_MODE`
  - 启动期直接失败
  - 不再允许“写错 mode 但静默落到 HTTP”

### QW

- `QW_MODE=MOCK`
  - 走应用内 `QwBenefitClientImpl` mock 分支
  - 适合轻量本地调试和无需真实上游的 focused 场景
- `QW_MODE=HTTP`
  - 走 `QwBenefitClientImpl` HTTP 分支
  - 当前与 `QWEIMOBILE_HTTP` 一样，都会路由到 qweimobile HTTP 实现
- `QW_MODE=QWEIMOBILE_HTTP`
  - 走 `QwBenefitClientImpl` HTTP 分支
  - 保留该 mode 主要是兼容现有配置口径
- `QW_MODE=ALLINPAY_DIRECT`
  - 借款权益主业务接口路由到 `AllinpayDirectQwBenefitClient`
  - 签约相关接口 `querySignStatus/applySign/confirmSign` 仍固定走 `QwBenefitClientImpl`
  - 这是当前真实行为，测试已锁定
- 非法 `QW_MODE`
  - 由 `QwProperties.Mode` 枚举绑定直接 fail-fast

## 推荐分工

### 只 mock Yunka

适用：

- 借款 / 还款链路只想本地走通业务代码
- 不需要 H5 全流程页面联调

后端启动示例：

```bash
mvn -Dspring-boot.run.arguments='--nexusfin.third-party.yunka.mode=MOCK --nexusfin.third-party.qw.mode=MOCK --nexusfin.auth.jwt.cookie-secure=false' spring-boot:run
```

说明：

- 不需要启动 `18081` Yunka stub
- `YUNKA_MODE=MOCK` 的返回是轻量空壳，不等价于 H5 happy-path 所需的完整字段

### 只 mock QW

适用：

- 权益侧只想跳过 QW 上游
- Yunka 仍需要按 REST 或本地 stub 口径联调

后端启动示例：

```bash
mvn -Dspring-boot.run.arguments='--nexusfin.third-party.yunka.mode=REST --nexusfin.third-party.yunka.base-url=http://127.0.0.1:18081 --nexusfin.third-party.qw.mode=MOCK --nexusfin.auth.jwt.cookie-secure=false' spring-boot:run
```

说明：

- 这时 Yunka 仍建议走 `REST + 18081`，QW 则完全由应用内 mock 承担

### 同时 mock Yunka + QW

适用：

- 只验证控制器/服务编排
- 不需要外部 happy-path 数据

后端启动示例：

```bash
mvn -Dspring-boot.run.arguments='--nexusfin.third-party.yunka.mode=MOCK --nexusfin.third-party.qw.mode=MOCK --nexusfin.auth.jwt.cookie-secure=false' spring-boot:run
```

说明：

- 这是最轻量的本地启动方式
- 不是 H5 全流程推荐模式

### H5 全流程联调

适用：

- 需要借款 / 审批 / 还款 happy-path 页面可走通

推荐口径：

- `YUNKA_MODE=REST`
- Yunka `base-url=http://127.0.0.1:18081`
- `QW_MODE=MOCK` 或按当前联调需要切到其它模式

启动参考：

- Yunka stub / tech-user stub / tmux 常驻口径见：
  - `docs/test/20260429_H5本地联调_stub启动说明.md`
  - `docs/test/20260429_H5全流程回归结果.md`

关键说明：

- `18081` 是“外部本地 Yunka stub”，只在 `YUNKA_MODE=REST` 下生效
- 它的职责是提供 H5 happy-path 所需的字段和最小状态推进
- 它不是 `YUNKA_MODE=MOCK` 的替代实现

## 当前日志口径

启动后可从日志直接判断当前模式：

- Yunka：
  - `yunka gateway client initialized enabled=... mode=... baseUrl=... gatewayPath=...`
- QW：
  - `qw benefit client routing initialized mode=... businessDelegate=... signDelegate=qweimobileClient`

这两条日志用于快速判断：

- 当前是否误跑在 `MOCK`
- QW 主业务请求当前会落到 `qweimobileClient` 还是 `allinpayDirectClient`

## 最小验证命令

```bash
mvn -Dtest=RestYunkaGatewayClientTest,RoutingQwBenefitClientTest,QwBenefitClientImplTest,AllinpayDirectQwBenefitClientTest test
mvn -q -DskipTests compile
```

## Layer B 故障注入最小口径

### 支持的 fault 标记

- `_FAULT_TIMEOUT`
- `_FAULT_REJECT_<code>`
- `_FAULT_DELAY_<ms>`

当前未实现：

- `_FAULT_EMPTY`
- `_FAULT_MALFORMED`
- `_FAULT_NETWORK`

### Yunka 本地 stub

适用模式：

- `YUNKA_MODE=REST`
- `base-url=http://127.0.0.1:18081`

触发优先级：

1. 外层 `requestId`
2. 外层 `bizOrderNo`
3. `payload.data` 内任意字符串字段

已覆盖 path：

- `/loan/apply`
- `/loan/query`
- `/repay/trial`
- `/card/smsSend`
- `/card/smsConfirm`
- `/repay/apply`
- `/repay/query`

示例：

- `platformBenefitOrderNo=BO-001_FAULT_TIMEOUT`：触发 `/loan/apply` 超时
- `loanId=LN-001_FAULT_REJECT_509`：触发还款类路径 reject
- `swiftNumber=RP-001_FAULT_DELAY_2000`：触发 `/repay/query` 延迟 2 秒

### QW 应用内 MOCK

适用模式：

- `QW_MODE=MOCK`

触发优先级：

1. 业务字段后缀
2. `X-Trace-Id` 兜底

已覆盖接口：

- `syncMemberOrder`
- `querySignStatus`
- `applySign`
- `confirmSign`
- `getExerciseUrl`
- `notifyLending`

推荐触发方式：

- `syncMemberOrder`：在 `partnerOrderNo` 带 fault 后缀
- `querySignStatus` / `applySign`：在 `accountNo` 带 fault 后缀
- `confirmSign`：优先使用 `X-Trace-Id: ..._FAULT_TIMEOUT`，或在 `verificationCode` 带 fault 后缀

示例：

```bash
curl -H 'X-Trace-Id: REQ_EX_QW_CONFIRM_FAULT_TIMEOUT' ...
curl -d '{"verificationCode":"123456_FAULT_REJECT_503", ...}' ...
curl -d '{"requestId":"req-1","productCode":"HUXUAN_CARD","loanAmount":800000,"agreementSigned":true}' ...
```

## 本任务明确不覆盖

- 科技平台登录真实联调
- 联合登录入口真实联调
- Yunka 上游复杂异常状态机
- 新增运行时动态切换接口
- 新建 QW 外部本地 stub 体系
