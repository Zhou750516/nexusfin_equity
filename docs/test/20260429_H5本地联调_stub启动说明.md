# 2026-04-29 H5 本地联调 stub 启动说明

## 目标

为 H5 本地全流程联调提供可复现的基础 stub，不再依赖历史 `/tmp/*.js`。

## 启动方式

### 科技平台用户信息 stub

```bash
cd /Users/lixiaokun/Projects/nexusFin/nexusfin-equity
node scripts/local-stubs/tech-user-stub.js
```

默认监听：

- `http://127.0.0.1:18080/api/users/me`

用途：

- 只服务登录链路
- 只接受 `Authorization: Bearer mock-tech-token`

### Yunka gateway stub

```bash
cd /Users/lixiaokun/Projects/nexusFin/nexusfin-equity
node scripts/local-stubs/yunka-stub.js
```

默认监听：

- `http://127.0.0.1:18081/api/gateway/proxy`

用途：

- 只服务 Yunka gateway
- 入口固定为 `POST /api/gateway/proxy`

支持的最小 happy path：

- `/loan/trail`
- `/loan/apply`
- `/loan/query`
- `/loan/repayPlan`
- `/repay/trial`
- `/repay/apply`
- `/repay/query`
- `/card/userCards`

说明：

- `/loan/query` 使用最小内存状态推进。
- 同一个 `loanId` 第一次查询返回 `7002 reviewing`，达到 `YUNKA_APPROVE_AFTER_QUERY_COUNT` 后返回 `7001 approved`。
- 默认阈值为 `2`，可通过环境变量覆盖。

## 后端联调启动口径

```bash
cd /Users/lixiaokun/Projects/nexusFin/nexusfin-equity
mvn -Dspring-boot.run.arguments='--spring.datasource.username=root --spring.datasource.password= --nexusfin.auth.jwt.cookie-secure=false --nexusfin.third-party.yunka.mode=REST --nexusfin.third-party.yunka.base-url=http://127.0.0.1:18081 --nexusfin.auth.tech-platform-base-url=http://127.0.0.1:18080' spring-boot:run
```

## 最小自检

```bash
curl -i http://127.0.0.1:18080/api/users/me -H 'Authorization: Bearer mock-tech-token'
curl -i http://127.0.0.1:18081/api/gateway/proxy
curl -i -c /tmp/nexusfin_auth_cookies.txt http://127.0.0.1:8080/api/auth/sso-callback?token=mock-tech-token
curl -i -b /tmp/nexusfin_auth_cookies.txt http://127.0.0.1:8080/api/users/me
```
