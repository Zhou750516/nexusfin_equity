# 2026-04-15 H5 本地联调启动与检查清单

## 1. 文档目的

用于固定 H5 页面联调时的本地启动口径，避免出现：

1. 后端虽然连接了 MySQL，但仍以默认 `YUNKA_MODE=MOCK` 运行；
2. H5 页面看似可打开，但审批结果、还款金额等链路出现“假成功 / 假异常”；
3. 因缺少产品或代扣协议等前置数据，导致页面流程无法完整跑通。

> 本清单适用于本地 H5 + 本地 Java 后端 + 本地 MySQL + 本地科技平台 stub + 本地云卡 stub 的联调场景。

---

## 2. 联调目标口径

本地联调的正确目标环境应为：

| 组件 | 正确口径 |
|---|---|
| H5 | `http://127.0.0.1:3000` |
| Java 后端 | `http://127.0.0.1:8080` |
| 数据库 | MySQL `nexusfin_equity` |
| 科技平台用户信息 | 本地 stub `http://127.0.0.1:18080/api/users/me` |
| 云卡 gateway | 本地 stub `http://127.0.0.1:18081/api/gateway/proxy` |
| 云卡模式 | `REST`，不是 `MOCK` |

## 3. 正确启动命令

### 3.1 启动 H5

```bash
cd /Users/lixiaokun/Projects/nexusFin/nexusfin-equity/H5
pnpm dev
```

期望：

- H5 可访问 `http://127.0.0.1:3000`

### 3.2 启动科技平台用户信息 stub

如果本地 stub 未启动，需要先确认 `18080` 可用。

期望：

- `http://127.0.0.1:18080/api/users/me` 可返回 mock 用户信息

### 3.3 启动云卡 gateway stub

当前本地云卡 stub 口径：

- 端口：`18081`
- 入口：`/api/gateway/proxy`

期望：

- `http://127.0.0.1:18081/api/gateway/proxy` 可访问

### 3.4 启动 Java 后端（正确口径）

**必须使用下面这条命令：**

```bash
cd /Users/lixiaokun/Projects/nexusFin/nexusfin-equity
mvn -Dspring-boot.run.arguments='--spring.datasource.username=root --spring.datasource.password= --nexusfin.auth.jwt.cookie-secure=false --nexusfin.third-party.yunka.mode=REST --nexusfin.third-party.yunka.base-url=http://127.0.0.1:18081' spring-boot:run
```

### 3.5 这条命令解决了什么

| 参数 | 作用 |
|---|---|
| `--spring.datasource.username=root` | 使用本地 MySQL root |
| `--spring.datasource.password=` | 使用空密码 root |
| `--nexusfin.auth.jwt.cookie-secure=false` | 允许本地 `http` 下写 Cookie |
| `--nexusfin.third-party.yunka.mode=REST` | 强制后端走本地云卡 REST stub |
| `--nexusfin.third-party.yunka.base-url=http://127.0.0.1:18081` | 指向本地云卡 stub |

> **最关键的是后两项。** 如果遗漏，后端会默认按 `application.yml` 走 `YUNKA_MODE=MOCK`，从而导致审批结果和还款金额异常。

## 4. 启动后第一时间检查项

### 4.1 端口检查

```bash
lsof -nP -iTCP:3000 -sTCP:LISTEN
lsof -nP -iTCP:8080 -sTCP:LISTEN
lsof -nP -iTCP:18080 -sTCP:LISTEN
lsof -nP -iTCP:18081 -sTCP:LISTEN
```

期望：

- `3000`：H5 在监听
- `8080`：Java 后端在监听
- `18080`：科技平台 stub 在监听
- `18081`：云卡 stub 在监听

### 4.2 后端日志检查

后端启动后，至少确认以下两点：

1. 成功连接 MySQL；
2. Tomcat 已监听 `8080`。

如果要进一步确认云卡口径，建议直接看启动命令，而不是只看日志。

> 原因：日志里不一定直接打印 `YUNKA_MODE`，但启动命令是否带 `--nexusfin.third-party.yunka.mode=REST` 是确定性的。

## 5. 联调前置数据检查清单

### 5.1 检查 `HUXUAN_CARD` 是否存在

```bash
mysql -uroot -D nexusfin_equity -e "SELECT * FROM benefit_product WHERE product_code='HUXUAN_CARD';"
```

期望：

- 至少返回 1 条 `HUXUAN_CARD` 产品记录

如果没有，补入：

```sql
INSERT INTO benefit_product (product_code, product_name, fee_rate, status, created_ts, updated_ts)
VALUES ('HUXUAN_CARD', '惠选卡', 300, 'ACTIVE', NOW(), NOW())
ON DUPLICATE KEY UPDATE
  product_name = VALUES(product_name),
  fee_rate = VALUES(fee_rate),
  status = VALUES(status),
  updated_ts = NOW();
```

### 5.2 准备登录态 Cookie

浏览器访问：

```text
http://127.0.0.1:8080/api/auth/sso-callback?token=mock-tech-token
```

期望：

- 后端能创建本地登录态 `NEXUSFIN_AUTH`
- 即使跳转到 `/equity/index` 404，也不影响 Cookie 已写入

### 5.3 准备代扣协议

需要调用签名保护的回调接口：

- `POST /api/callbacks/payment-protocol`

建议沿用浏览器同源脚本或已有本地回调方式，确保为当前用户补入一条 `ACTIVE` 协议。

建议检查：

```bash
mysql -uroot -D nexusfin_equity -e "SELECT id, member_id, external_user_id, provider_code, protocol_no, protocol_status, created_ts FROM member_payment_protocol ORDER BY id DESC LIMIT 5;"
```

期望：

- 当前联调用户存在一条 `ACTIVE` 的支付协议

## 8. 最容易犯错的点

### 错误 1：只切到 MySQL，没有切到云卡 REST

**错误现象**

- `loan/apply` 看起来成功
- `approval-status` 一直 `reviewing`
- `approval-result` 变成 `rejected`
- `repayment/info` 金额是 `0.00`

**根因**

- 后端仍然是 `YUNKA_MODE=MOCK`

**避免方式**

- 启动后端时必须带：
  - `--nexusfin.third-party.yunka.mode=REST`
  - `--nexusfin.third-party.yunka.base-url=http://127.0.0.1:18081`

### 错误 2：没有预置 `HUXUAN_CARD`

**错误现象**

- 首页点击“确认借款”后返回：
  - `PRODUCT_NOT_FOUND:Benefit product not found`

**避免方式**

- 联调前先检查 `benefit_product` 是否有 `HUXUAN_CARD`

### 错误 3：没有准备有效支付协议

**错误现象**

- 借款申请或权益开通链路无法正常前进

**避免方式**

- 在联调前先补一条当前用户的 `ACTIVE` 支付协议

### 错误 4：本地 Cookie 仍按 `secure=true`

**错误现象**

- SSO 看似成功，但 H5 接口仍未登录

**避免方式**

- 后端启动时必须带：
  - `--nexusfin.auth.jwt.cookie-secure=false`

## 9. 推荐执行顺序

建议每次本地联调都按下面顺序执行：

1. 启动 MySQL
2. 启动科技平台 stub
3. 启动云卡 stub
4. 启动 H5
5. 用“正确启动命令”启动 Java 后端
6. 检查 `HUXUAN_CARD`
7. 浏览器打 SSO 登录
8. 补支付协议
9. 从首页走主流程
10. 用 SQL 验证关键落库结果

---

## 10. 一句话标准口径

> **本地 H5 联调必须同时满足：MySQL 已连接、云卡已切到 REST stub、`HUXUAN_CARD` 已存在、支付协议已补入。缺任何一项，页面链路都可能出现假异常。**
