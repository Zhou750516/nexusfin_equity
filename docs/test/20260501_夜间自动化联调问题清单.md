## P1-3 H5 `benefits-card` 开通按钮在普通 SSO 借款链下无法拿到 token

状态：`PASS（2026-05-01 Round 3 已修复并回归通过）`

严重度判断：

1. 页面能打开，不是白屏。
2. `benefits/activate` 后端接口本身已经恢复可用。
3. 但普通借款主链进来的用户，点“立即开通”会在前端本地失败，导致权益开通入口实际不可用。
4. 因此按联调口径记录为 `P1 页面接线缺陷`。

运行态证据：

1. 使用普通 SSO 登录用户 `tech-user-local-001` 打开：

```text
/benefits-card?applicationId=APP-night-activate-002
```

2. 页面正常展示协议、默认卡、权益信息。
3. 点击“立即开通”后，页面直接出现：

```text
加载失败
缺少联登凭证，暂无法生成权益跳转链接。
```

最小根因：

1. `ApprovalPendingPage` 导航到 `/benefits-card` 只带了 `applicationId`
2. `BenefitsCardPage.handleActivate()` 强依赖 `readJointLoginParams()?.token`
3. 普通借款 / SSO 主链不会天然具备 joint-login sessionStorage

代码证据：

```text
H5/client/src/pages/ApprovalPendingPage.tsx
  const benefitsPath = applicationId
    ? buildPath("/benefits-card", { applicationId })
    : "/benefits-card";

H5/client/src/pages/BenefitsCardPage.tsx
  const jointLoginToken = readJointLoginParams()?.token;
  if (!jointLoginToken) {
    setError(MISSING_JOINT_TOKEN_COPY[locale]);
    return;
  }
```

影响：

1. 这会让“页面展示通过”和“真实可开通”出现语义分裂。
2. 不能把当前 H5 权益页宣称为主链已闭环。

修复动作：

1. `ApprovalPendingPage` 只有在当前会话存在 joint-login token 时才展示权益入口。
2. `BenefitsCardPage` 把 `hasJointLoginToken` 纳入 CTA enable 条件。
3. 无 token 时页面直接展示前置提示，而不是点击后才报 `加载失败`。

回归证据：

1. `cd H5 && ./node_modules/.bin/vitest run client/src/pages/benefits-card.logic.test.ts client/src/pages/approval-pending-benefits.logic.test.ts`
   - `10 passed`
2. `cd H5 && ./node_modules/.bin/tsc --noEmit`
   - `exit code 0`
3. 浏览器清空 `sessionStorage['nexusfin.h5.joint-login.params']` 后重开 `/benefits-card?applicationId=APP-night-001`
   - CTA 已禁用
   - 页面展示 `缺少联登凭证，暂无法生成权益跳转链接。`
   - 不再出现点击后本地失败态
