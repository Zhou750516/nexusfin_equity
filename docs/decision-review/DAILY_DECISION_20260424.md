### DECISION-056: 下一阶段按“共享基础 → 权益页动态化 → 借款/还款对齐”顺序推进

**决策日期**: 2026-04-24  
**决策人**: 技术负责人 + 开发负责人  
**所属分类**: 开发顺序 / 交付路径  
**优先级**: ⭐⭐⭐ 高  
**影响范围**: 大

#### 问题背景

当前缺口分散在 Yunka 路径配置、typed DTO、facade、权益协议与绑卡、借款状态映射、还款短信确认、回调模型等多个方向。

如果没有顺序约束，会出现：

1. service 继续各自拼 JSON，后续再统一成本更高
2. 借款与还款并行改造时模型不一致
3. 权益页和回调页字段语义继续分裂

#### 最终决策

**选择**: 先收口共享基础，再做权益页动态化，再推进借款 / 还款链路对齐。

#### 理由

1. Shared Foundation 是所有后续故事的阻塞前置
2. 权益页动态化相对借款主链重构风险更低，且收益更直接
3. 借款和还款链路都依赖统一模型、统一路径和统一回调语义

#### 实施结果

- [x] 形成阶段 A-F 的任务分组
- [x] 明确 Shared Foundation 为阻塞前置
- [x] 明确权益页动态化优先于借款主链重构
