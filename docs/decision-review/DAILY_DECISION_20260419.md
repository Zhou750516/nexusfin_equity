### DECISION-041: 控制器级返回口径继续区分“权益超时失败”和“用信超时审核中”

**决策日期**: 2026-04-19  
**决策人**: 技术负责人 + 开发负责人  
**所属分类**: 业务口径 / 接口承接  
**优先级**: ⭐⭐⭐ 高  
**影响范围**: 大

#### 问题背景

异步补偿机制接入后，容易出现一个误区：只要后端进了补偿，就都应该返回“处理中”。

但当前已确认业务口径并不是统一如此：

1. 权益购买同步超时 / 报错：当前仍按同步失败承接
2. 权益购买成功后，用信提交超时：当前按“已提交 / 审核中”承接
3. 权益购买成功后，用信明确失败：当前按 `loan_failed` 承接，并明确权益已成功

#### 最终决策

**选择**: 保持当前三类口径分开，不因为“都有补偿”就强行统一前端返回

**理由**:

- 业务确认只要求“用信提交超时”场景前端展示“已提交 / 审核中”。
- 权益购买同步超时场景，目前只要求保留补偿任务，并未确认要改成“处理中”。
- 继续保持三类语义分开，能避免前端和业务对状态理解混乱。

#### 下步行动

- [x] 输出控制器级返回口径核对结论
- [x] 明确当前已一致与待继续观察的点
- [ ] 若后续业务要求权益购买超时也展示处理中，再单独补语义调整

### DECISION-042: 修复权益购买超时场景的事务回滚，确保补偿数据不丢失

**决策日期**: 2026-04-19  
**决策人**: 技术负责人 + 开发负责人  
**所属分类**: 技术修复 / 可靠性  
**优先级**: ⭐⭐⭐ 高  
**影响范围**: 大

#### 问题背景

在今天的控制器级口径核对中，新增集成测试发现：

1. `loan/apply` 触发齐为权益购买同步超时时
2. 代码虽然会尝试更新权益订单状态并写入补偿任务
3. 但由于外层事务回滚，最终权益订单和补偿任务都会一起丢失

这会导致“接口返回失败，但本地无补偿证据链”，与当前补偿设计目标不一致。

#### 最终决策

**选择**: 通过专用异常 + 关闭特定异常事务回滚的方式做最小修复

**理由**:

- 根因不在补偿任务本身，而在事务语义。
- 最小修复即可保留当前业务口径，不需要改控制器返回语义。
- 能确保“同步失败，但补偿数据保留”的关键目标成立。

#### 实施结果

- [x] 新增 `BenefitPurchaseSyncTimeoutCompensationException`
- [x] `BenefitOrderServiceImpl.createOrder()` 对该异常关闭事务回滚
- [x] `LoanServiceImpl.apply()` 对该异常关闭事务回滚
- [x] 新增 `Phase9TaskGroupECompensationIntegrationTest`
- [x] 回归通过：
  - `Phase9TaskGroupECompensationIntegrationTest`
  - `Phase9TaskGroupEIntegrationTest`
  - `BenefitOrderServiceTest`

### DECISION-043: 自动调度第一阶段采用“默认关闭、显式开启”

**决策日期**: 2026-04-19  
**决策人**: 技术负责人 + 开发负责人  
**所属分类**: 调度接入 / 上线控制  
**优先级**: ⭐⭐⭐ 高  
**影响范围**: 大

#### 问题背景

异步补偿已经补齐 `worker / supervisor / scheduler` 入口后，如果默认即自动运行，会带来两个风险：

1. 污染现有联调与测试环境
2. 在未显式配置分片归属前，容易让协作方误以为“已经进入正式常驻运行”

因此需要明确第一阶段的启用策略。

#### 最终决策

**选择**: 第一阶段采用“默认关闭、显式开启、显式分片”的接入策略

**理由**:

- 当前目标是把自动调度从“方案”推进到“可上线骨架”，不是立即在所有环境自动跑起来。
- 默认关闭可以保证既有 Phase 9、H5 联调和现有集成测试语义不被悄悄改变。
- 显式 `workerId` / `ownedPartitions` 更利于问题排查和实例职责确认。

#### 实施结果

- [x] `AsyncCompensationProperties` 新增调度开关与分片配置项
- [x] `NexusfinEquityApplication` 启用 `@EnableScheduling`
- [x] 新增 `AsyncCompensationWorkerScheduler`
- [x] 新增 `AsyncCompensationSupervisorScheduler`
- [x] 应用启动打印调度摘要日志
- [x] 验证默认关闭时 Spring Boot 启动正常
- [x] 验证默认关闭时，权益购买超时场景仅落 `INIT` 任务，不产生 heartbeat / attempt 副作用
