# 阿里云齐为测试环境配置说明

本文用于部署会话将齐为从 MOCK 模式切换为真实齐为测试环境 HTTP 调用。密钥值只从安全配置或环境变量注入，不写入代码、默认配置或日志。

## 环境变量

| 变量名 | 建议值 | 含义 | 来源 |
| --- | --- | --- | --- |
| `QW_ENABLED` | `true` | 启用齐为集成 | 部署配置 |
| `QW_MODE` | `HTTP` | 使用齐为 HTTP client，不走 MOCK，不走盛付通 direct | 部署配置 |
| `QW_BASE_URL` | `https://t-api.test.qweimobile.com` | 齐为测试环境域名 | `艾博生-齐为测试环境.txt` |
| `QW_METHOD_PATH` | `/api/abs/method` | 齐为统一 method 入口路径 | `艾博生-齐为测试环境.txt` |
| `QW_PARTNER_NO` | `abs` | 齐为分配的合作方编号 | `艾博生-齐为测试环境.txt` |
| `QW_VERSION` | `v1.0` | 当前代码默认协议版本；如齐为另有确认，以齐为确认为准 | 当前协议配置 |
| `QW_SIGN_KEY` | 从安全配置注入 | MD5 签名密钥，禁止明文打印 | `艾博生-齐为测试环境.txt` |
| `QW_AES_KEY` | 从安全配置注入 | AES 密钥，禁止明文打印 | `艾博生-齐为测试环境.txt` |
| `QW_AES_KEY_ENCODING` | `RAW` | AES key 按原始字符串读取 | 当前 HTTP client 实现 |
| `QW_AES_ALGORITHM` | `AES/ECB/PKCS5Padding` | 齐为报文加解密算法 | 当前 HTTP client 实现 |
| `QW_CIPHERTEXT_ENCODING` | `HEX` | 加密报文使用 hex 编码 | 当前 HTTP client 实现 |
| `LOG_LEVEL_QW` | `INFO` | 打开齐为关键运行日志 | 部署配置 |
| `H5_BENEFITS_PRODUCT_CODE` | `abs001` | 测试环境可购买会员卡产品码 | `艾博生-齐为测试环境.txt` |

## 启动校验

当 `QW_MODE=HTTP` 或 `QW_MODE=QWEIMOBILE_HTTP` 时，backend 启动期会校验以下配置：

- `QW_BASE_URL` 非空
- `QW_METHOD_PATH` 非空
- `QW_PARTNER_NO` 非空
- `QW_VERSION` 非空
- `QW_SIGN_KEY` 非空，且不能继续使用默认占位值
- `QW_AES_KEY` 或 `QW_AES_KEY_BASE64` 非空，且不能继续使用默认占位值
- `QW_AES_ALGORITHM` 非空
- `QW_CIPHERTEXT_ENCODING` 非空

如果缺少真实密钥或仍使用默认占位值，启动会以 `QW_HTTP_CONFIG_INVALID` 受控失败，避免云上误以为已经关闭 MOCK 但实际使用无效密钥。

## 启动日志确认

重启 backend 后，齐为路由初始化日志应包含类似字段：

```text
qw runtime configuration qw_mode=HTTP qw_base_url=https://t-api.test.qweimobile.com qw_method_path=/api/abs/method qw_partner_no=abs qw_sign_key=configured qw_aes_key=configured businessDelegate=qweimobileClient signDelegate=qweimobileClient
```

确认点：

- `qw_mode=HTTP`
- `businessDelegate=qweimobileClient`
- 不应出现 `qw_mode=MOCK`
- 不应出现 `businessDelegate=allinpayDirectClient`
- `qw_sign_key` 和 `qw_aes_key` 只能显示 `configured` 或 `missing`，不能显示明文

## 运行态日志确认

触发齐为 HTTP 调用后，运行日志应包含类似字段：

```text
qw http request prepared qw_mode=HTTP qw_target_uri=https://t-api.test.qweimobile.com/api/abs/method qw_method=abs.deduction.notify qw_partner_no=abs
```

重点确认：

- `qw_target_uri=https://t-api.test.qweimobile.com/api/abs/method`
- `qw_method` 为当前业务动作对应 method，例如 `abs.push.order`、`abs.token.get`、`abs.deduction.notify`、`abs.deduction.query`、`abs.order.cancel`
- 日志不包含 `QW_SIGN_KEY`、`QW_AES_KEY` 的明文
- 日志不打印加密前业务明文

## 部署步骤建议

1. 部署包含齐为 HTTP 配置校验和脱敏日志的新 backend 版本。
2. 在 `.env.aliyun` 或启动环境中设置上表环境变量。
3. 确认 `QW_MODE=HTTP`，不要设置为 `MOCK` 或 `ALLINPAY_DIRECT`。
4. 重启 backend。
5. 通过启动日志确认 `qw_mode=HTTP` 且 `businessDelegate=qweimobileClient`。
6. 触发一次齐为链路，通过运行态日志确认请求目标为 `https://t-api.test.qweimobile.com/api/abs/method`。
