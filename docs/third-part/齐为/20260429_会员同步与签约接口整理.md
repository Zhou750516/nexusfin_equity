## 2. 公共协议说明

### 2.1 接口调用地址

| 项 | 内容 |
|---|---|
| 测试地址 | `https://t-api.test.qweimobile.com/api/abs/method` |
| 网络协议 | `HTTPS` |
| Content-Type | `application/json` |
| 字符集 | `UTF-8` |

### 2.2 公共请求结构

```json
{
  "requestBody": "AES加密后的业务数据",
  "requestHead": {
    "partnerNo": "appid",
    "timestamp": "1655891248477",
    "method": "接口方法名",
    "sign": "md5签名结果",
    "version": "V1.0"
  }
}
```

### 2.3 公共请求参数说明

| 参数 | 位置 | 是否必填 | 说明 |
|---|---|---|---|
| `requestBody` | body | 是 | 使用 AES 加密之后的业务数据，加密源为 String 类型 JSON |
| `partnerNo` | `requestHead` | 是 | 商户编号 |
| `timestamp` | `requestHead` | 是 | 时间戳 |
| `method` | `requestHead` | 是 | 参考接口 method 枚举 |
| `version` | `requestHead` | 是 | 固定值：`v1.0` |
| `sign` | `requestHead` | 是 | `Md5(String.format("%s%s%s%s%s", method, partnerNo, timestamp, version, key))` |

### 2.4 公共返回结构

```json
{
  "msg": "成功",
  "code": 200,
  "data": "AES加密后的业务数据"
}
```

### 2.5 公共返回参数说明

| 参数 | 是否必填 | 说明 |
|---|---|---|
| `msg` | 是 | 响应结果中文描述 |
| `code` | 是 | 成功：`200`，失败：`500` |
| `data` | 否 | 使用 AES 加密之后的业务数据 |

### 2.6 method 枚举

| method | 描述 |
|---|---|
| `abs.push.order` | 会员同步接口 |
| `abs.sign.query` | 签约查询 |
| `abs.sign.apply` | 签约申请 |
| `abs.sign.confirm` | 签约确认 |

## 3. 会员同步接口

### 3.1 基本信息

| 项 | 内容 |
|---|---|
| 接口名称 | 会员同步接口 |
| method | `abs.push.order` |
| 接口说明 | 合作方同步会员信息到权益平台 |

### 3.2 业务请求参数

| 参数名 | 类型 | 名称 | 是否必填 | 说明 |
|---|---|---|---|---|
| `uniqueId` | `string` | 用户ID | 是 | 会员用户唯一标识 |
| `partnerOrderNo` | `string` | 合作方订单号 | 是 | 流水号，保证唯一性 |
| `payAmount` | `int` | 支付金额 | 是 | 支付金额，单位分；扣款要扣的费用 |
| `productCode` | `string` | 产品编码 | 是 | 产品编码 |
| `productName` | `string` | 产品名称 | 否 | 产品名称，可不填 |
| `mobile` | `string` | 会员手机号 | 是 | 签约的手机号 |
| `username` | `string` | 用户姓名 | 是 | 签约的用户名 |
| `accountNo` | `string` | 银行卡号 | 是 | 签约的银行卡号 |
| `cardNo` | `string` | 会员卡号 | 否 | 为空则视为新卡，否则认为续期，`issue > 1` 时必填 |
| `shareFlag` | `int` | 清分标识 | 是 | `0` 否，`1` 是 |
| `partnerMerchantNo` | `string` | 权益平台商户号 | 否 | `shareFlag=1` 时必填 |
| `partnerShareAmount` | `int` | 权益平台清分金额 | 否 | `shareFlag=1` 时必填，单位分 |
| `shareMerchantNo` | `string` | 合作方商户号 | 否 | `shareFlag=1` 时必填 |
| `shareAmount` | `int` | 合作方清分金额 | 否 | `shareFlag=1` 时必填 |

### 3.3 业务返回参数

| 参数名 | 类型 | 名称 | 是否必填 | 说明 |
|---|---|---|---|---|
| `orderNo` | `string` | 系统订单号 | 是 | 系统订单号 |
| `cardNo` | `string` | 用户卡号 | 是 | 用户卡号 |
| `timestamp` | `string` | 时间戳 | 是 | 时间戳 |
| `openFlag` | `int` | 开卡结果 | 是 | 成功：`0`；退款：`1`；开通中：`2` |
| `productCode` | `string` | 产品编码 | 是 | 对应产品编码，商务确认 |
| `productName` | `string` | 产品名称 | 是 | 产品名称 |
| `productType` | `string` | 产品类型 | 是 | `independence` 多卡独立；`renew` 单卡续费 |
| `cardCreatedDate` | `string` | 开通日期 | 否 | `yyyy-MM-dd HH:mm:ss` |
| `cardExpiryDate` | `string` | 失效日期 | 否 | `yyyy-MM-dd HH:mm:ss` |

### 3.4 本次更新点

根据 `2026-04-21` 版本文档，本接口新增 / 明确了以下字段：

1. `mobile`
2. `username`
3. `accountNo`
4. `cardNo`

### 7.3 当前已确认口径补充

根据当前最新确认，补充以下执行口径：

#### A. 返回结构口径

签约查询、签约申请、签约确认接口的外层返回结构统一参考齐为公共返回结构：

```json
{
  "msg": "成功",
  "code": 200,
  "data": "AES加密后的业务数据"
}
```

失败时：

1. 返回结构不变；
2. `code` 会变化；
3. `data` 会变化；
4. 具体失败码与业务 `data` 结构，后续再结合齐为正式联调结果补充。

#### B. 签约号口径

本次不要求艾博生侧显式使用签约号 / 协议号 / 申请流水号作为页面或后端接口编排依赖。

当前口径：

1. 签约号在齐为内部传递；
2. 之前 mock 方案中的签约号占位，本次不再作为正式依赖；
3. 本侧当前优先保证签约成功结果能够被识别和记录。

签约成功后，艾博生侧需要本地记录签约结果。

3. 记录时点以“签约确认成功”为准；
4. 后续查询接口是否以本地记录作为辅助判断依据。
