# Loan App — Codex 开发交接文档

> **项目名称：** loan-app（金融贷款应用）
> **技术栈：** React 19 + TypeScript 5.6 + Tailwind CSS 4 + Vite 7 + wouter 路由
> **当前状态：** 纯前端静态项目，6 个页面 UI 已还原，所有业务数据为硬编码 Mock
> **文档版本：** v1.0 | 2026-04-12
> **作者：** Manus AI

---

## 一、项目概述

本项目是一个移动端金融贷款应用的前端实现，基于 Figma 设计稿严格还原了 6 个核心业务页面。目前所有页面均为纯前端静态展示，业务数据（借款金额、利率、还款计划等）全部硬编码在组件内部。如需继续使用 Codex 进行完善优化，需要将项目导出并补充后端接口，使其成为可真正运行的业务系统。

### 业务流程

```
试算页 → 用信审批页 → 惠选卡页 → 审批结果页 → 确认还款页 → 还款成功页
```

用户在试算页选择借款金额和期数后确认借款，进入审批等待页面；审批期间可选择开通惠选卡增值服务；审批通过后查看结果，进入还款流程，最终完成还款。

## 三、需要补充的后端接口

当前项目为纯前端，所有业务数据均为硬编码。以下是按业务流程梳理的完整后端接口清单，分为**核心接口**（必须实现）和**增强接口**（建议实现）两类。

### 3.1 核心接口一览

| 序号 | 接口名称 | 方法 | 路径 | 关联页面 |
|---|---|---|---|---|
| 1 | 获取试算参数 | GET | `/api/loan/calculator-config` | 试算页 |
| 2 | 提交试算请求 | POST | `/api/loan/calculate` | 试算页 |
| 3 | 提交借款申请 | POST | `/api/loan/apply` | 试算页 → 审批页 |
| 4 | 查询审批状态 | GET | `/api/loan/approval-status/:id` | 审批页 |
| 5 | 获取惠选卡详情 | GET | `/api/benefits/card-detail` | 惠选卡页 |
| 6 | 开通惠选卡 | POST | `/api/benefits/activate` | 惠选卡页 |

### 4.2 建议新增的前端基础设施

在进行接口对接之前，建议先搭建以下基础设施代码：

**HTTP 请求封装（`client/src/lib/api.ts`）：** 基于已安装的 `axios`，封装统一的请求拦截器、响应拦截器、错误处理和 Token 注入逻辑。

```typescript
// 建议结构
import axios from "axios";

const api = axios.create({
  baseURL: "/api",
  timeout: 15000,
  headers: { "Content-Type": "application/json" },
});

// 请求拦截器：注入 Token
api.interceptors.request.use((config) => {
  const token = localStorage.getItem("auth_token");
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// 响应拦截器：统一错误处理
api.interceptors.response.use(
  (res) => res.data,
  (err) => {
    if (err.response?.status === 401) {
      // 跳转登录
    }
    return Promise.reject(err);
  }
);

export default api;
```

**全局状态管理（`client/src/contexts/LoanContext.tsx`）：** 使用 React Context 管理跨页面共享的贷款申请状态（applicationId、借款金额、期数等），避免通过 URL 参数传递大量数据。

**Loading/Error 状态组件：** 为接口请求添加统一的加载骨架屏和错误重试 UI。

pnpm install    # 安装依赖

## 七、安全合规注意事项

金融贷款应用涉及敏感数据和资金操作，在后续开发中必须重视以下安全要求：

**数据安全方面：** 所有接口必须使用 HTTPS 传输；用户身份证号、银行卡号等敏感字段在数据库中必须加密存储；前端展示时需进行脱敏处理（如银行卡号仅显示后四位）；接口响应中不应返回完整的敏感信息。
