## 2. 后续待办接口

### TODO 1：查看银行卡是否已签约

**目标**

- 在进入绑卡流程前，先判断当前银行卡是否已经完成签约 / 绑卡

**待确认内容**

- 接口路径
- 请求方式（`GET` / `POST`）
- 入参口径（用户标识、银行卡号、证件号、手机号等）
- 返回口径（已签约 / 未签约 / 签约中 / 失效）
- 是否需要幂等或缓存策略

### TODO 2：申请绑卡（下发短信）

**目标**

- 当银行卡未签约时，调用申请绑卡接口，由齐为或支付侧下发短信验证码

**待确认内容**

- 接口路径
- 请求方式
- 入参字段（持卡人姓名、身份证、银行卡号、预留手机号等）
- 返回字段（申请流水号、验证码有效期、是否需要图形验证码等）
- 重发短信规则
- 错误码口径

### TODO 3：绑卡确认（短信验证）

**目标**

- 用户输入短信验证码后，调用绑卡确认接口完成银行卡签约 / 绑卡

**待确认内容**

- 接口路径
- 请求方式
- 入参字段（申请流水号、短信验证码、用户标识等）
- 返回字段（签约号、绑卡状态、失败原因等）
- 成功后的落库字段
- 与现有支付协议 / 代扣协议表的映射关系
