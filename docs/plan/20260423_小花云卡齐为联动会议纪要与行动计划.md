# 2026-04-23 小花 / 云卡 / 齐为联动会议纪要与行动计划

## 1. 文档目标

本文档用于沉淀 `2026-04-23` 关于艾博生 H5、云卡、小花科技、齐为权益与法大大电子签章联动链路的最新会议结论，并拆解为后续可执行 plan。

本次会议重点包括：

1. 新增用信拒绝落地页
2. 试算页金额动态调整与综合年化成本确认
3. 拒量变现 / 助贷业务 / 借款协议查询路径区分
4. 用信提交链路调整为“云卡先建放款订单，权益购买后再放款”
5. 权益签约、短信签约、loading 状态、放弃权益与关单拒绝
6. 齐为签约成功后触发法大大电子签章
7. 云卡新增用信状态查询接口
8. 用户行权分发页
9. 已购买权益但借据未结清时的借款限制
10. H5 自动埋点方案

## 2. 会议结论

### 2.1 艾博生新增落地页，用于用信拒绝导流

本次会议确认：

1. 艾博生侧需要新增一个落地页
2. 用户用信被拒绝后，导流到该落地页

当前含义：

1. H5 页面范围需要新增“拒绝导流落地页”
2. 后端需要支持该页面所需的数据接口，或先由前端静态配置承接
3. 后续需要明确落地页目标：
   - 展示拒绝结果
   - 引导权益 / 其他产品
   - 引导小花或其他分发页

### 2.2 试算页金额支持动态调整，范围为 `100-5000`

本次会议确认：

1. 试算页当前默认 `3000`
2. 需要支持用户动态调整金额
3. 金额范围为 `100-5000`
4. 综合年化成本需要和小花科技确认

当前含义：

1. `calculator-config` 本地配置需要支持金额范围配置
2. H5 试算页需要支持金额滑动 / 输入调整
3. `/api/loan/calculate` 需要按动态金额调用云卡 / 小花试算
4. 综合年化成本不能自行硬编码，需待小花确认口径

### 2.3 查询路径区分：拒量变现、助贷业务、借款协议

本次会议确认：

| 场景 | 查询路径 | 当前口径 |
|---|---|---|
| 拒量变现环节 | 艾博生直接查询小花科技 | 不走云卡 |
| 助贷业务环节 | 艾博生通过云卡查询小花科技 | 走云卡透传 |
| 借款相关协议 | 通过云卡查询小花科技 | 走云卡 |

补充要求：

1. 借款相关协议需要考虑强制阅读
2. 用户需要阅读完整协议

当前含义：

1. 协议查询能力需要纳入云卡接口设计
2. H5 需要支持协议强制阅读状态
3. 后端需要考虑记录协议阅读完成状态，至少要能防止“未阅读完整直接提交”

### 2.4 用信提交链路调整：云卡先创建放款订单，权益购买后再放款

本次会议确认：

1. 艾博生用信提交时，云卡先创建放款订单
2. 云卡先不提交小花科技
3. 等艾博生完成权益购买后，再调用云卡放款
4. 云卡再调用小花科技平台

云卡侧需要设计：

1. 放款订单状态机
2. 关单逻辑

责任人：

- `@周老师`

当前含义：

1. 原先“艾博生用信提交 -> 云卡透传小花用信”需要调整
2. 新链路应拆成两段：
   - 阶段一：创建云卡放款订单
   - 阶段二：权益购买成功后触发云卡放款
3. 艾博生侧需要保存云卡放款订单号 / 状态映射
4. 现有异步补偿和日志链路需要重新评估触发点

### 2.5 权益开通需要短信签约，签约页到审批页需要 loading

本次会议确认：

1. 艾博生开通权益服务需要有短信签约
2. 从签约页面跳转到借款审批页面时，需要有 loading 状态

当前含义：

1. H5 后续需要新增或接入签约页面
2. 后端已具备齐为签约三接口骨架，但需要和真实 H5 页面联通
3. 签约成功后的跳转不能直接给用户“空白等待”，需要 loading 承接

### 2.6 用户放弃权益时，艾博生通知云卡，云卡判断是否关单拒绝

本次会议确认：

1. 当用户在艾博生侧放弃权益时，艾博生需要调用云卡
2. 云卡判断是否可以关单拒绝

当前含义：

1. 云卡需要提供“放弃权益 / 放弃用信 / 关单判断”类接口
2. 艾博生侧需要在 H5 放弃权益动作上补后端调用
3. 后续要明确：
   - 放弃权益是否一定导致拒绝
   - 云卡返回哪些状态
   - 艾博生页面如何承接

### 2.7 齐为签约成功后，艾博生后端需要调用法大大电子签章

本次会议确认：

1. 齐为返回签约成功后
2. 艾博生后端需要调用法大大做电子签章

当前含义：

1. 齐为签约成功不再只是本地记录协议
2. 还需要接续法大大签章链路
3. 法大大当前暂停开发，但方案和任务需要记录为后续关键链路
4. 后续需要明确：
   - 签章触发时机
   - 签章合同模板
   - 签章结果回调
   - 签章失败是否阻断借款审批

### 2.8 云卡需要提供用信状态查询接口，并区分小花用信结果查询

本次会议确认：

1. 云卡需要提供一个查询用信状态的接口
2. 该接口要和小花的用信结果查询区分开

责任人：

- `@周老师`

当前含义：

1. 艾博生查询审批状态时，不应直接等同于“小花用信结果查询”
2. 云卡侧可能需要维护自己的订单状态机
3. 艾博生应查询云卡用信状态，再由云卡内部决定是否查询 / 映射小花结果

### 2.9 用户行权：艾博生提供 Index 分发页

本次会议确认：

1. 用户行权时，艾博生提供一个 Index 分发页面
2. 用户可以从小花跳转到艾博生权益页面

当前含义：

1. 艾博生需要提供稳定分发页地址
2. 分发页需要根据参数识别用户、权益、订单或来源
3. 需要和前期“权益订单页 / 行权展示 / 退款跳转”口径统一

### 2.10 已购买权益但借据未结清时，不允许再次借款

本次会议确认：

1. 用户已经购买权益，但没有完成借据结清
2. 艾博生后端需要设置规则，不允许用户再次借款
3. 本期暂不考虑“一笔订单未完成，退出后再借另一笔”的场景

当前含义：

1. 后端需要在借款前增加资格 / 状态校验
2. 校验条件需要关联：
   - 权益订单状态
   - 借据状态
   - 是否已结清
3. 本期策略偏保守：未结清则阻断再次借款

### 2.11 需要设计 H5 自动埋点方案

本次会议确认：

1. 需要设计前端 H5 自动埋点方案

当前含义：

1. H5 页面需要统一采集页面访问、点击、流程状态等事件
2. 需要明确埋点范围、事件命名、上报接口、失败重试和隐私边界
3. 建议先形成方案，不立即重构全部页面

## 4. 行动计划

### P0：重构用信提交链路方案

**目标**

- 将原“提交用信即透传小花”调整为“云卡先建放款订单，权益成功后再放款”

**行动项**

1. 等云卡补放款订单状态机和关单逻辑
2. 艾博生侧设计本地放款订单映射字段
3. 调整 `/api/loan/apply` 后端流程：
   - 创建云卡放款订单
   - 引导权益购买 / 签约
   - 权益购买成功后触发云卡放款
4. 重新定义失败 / 超时 / 补偿触发点
5. 回填接口文档和 H5 流程说明

### P0：补齐权益短信签约与审批 loading 流程

**目标**

- 让权益签约成功到借款审批之间有明确页面承接和后端状态承接

**行动项**

1. 对接 H5 签约页设计
2. 复用齐为签约三接口：
   - 签约查询
   - 签约申请
   - 签约确认
3. 签约成功后记录本地协议状态
4. 签约成功后触发后续审批 loading
5. 明确 loading 页面轮询的是云卡用信状态还是艾博生本地状态

### P0：设计云卡用信状态查询对接

**目标**

- 区分“云卡用信状态”和“小花用信结果”

**行动项**

1. 等云卡提供接口定义
2. 明确状态枚举：
   - 订单已创建
   - 待权益购买
   - 待放款
   - 放款中
   - 成功
   - 拒绝
   - 关闭
3. 艾博生后端映射 H5 页面状态
4. 更新 approval-status / approval-result 相关接口设计

### P1：试算页动态金额与协议强制阅读

**目标**

- 提升试算页交互能力，并满足协议合规阅读要求

**行动项**

1. `calculator-config` 支持 `minAmount=100`、`maxAmount=5000`、默认 `3000`
2. H5 试算页支持动态调整
3. `/api/loan/calculate` 按动态金额重新试算
4. 等小花确认综合年化成本口径
5. 设计协议查询与完整阅读校验：
   - 协议列表查询
   - 协议内容展示
   - 阅读到底部后允许勾选 / 提交
   - 后端记录阅读状态

### P1：用信拒绝落地页与放弃权益关单

**目标**

- 补齐用信拒绝和用户放弃权益两个异常分支页面与后端链路

**行动项**

1. 设计用信拒绝落地页
2. 明确拒绝页入口参数和展示内容
3. 等云卡提供放弃权益 / 关单判断接口

5. 艾博生后端调用云卡通知放弃权益

### P1：行权 Index 分发页

**目标**

**行动项**

4. H5 根据上下文跳转权益页、退款页或其他页面

### P2：齐为签约后接法大大电子签章

## 6. 当前建议结论

截至 `2026-04-23`，本次会议对原有流程影响较大，尤其是：

1. **用信提交流程发生关键调整**：从“直接透传小花”调整为“云卡先建订单，权益成功后再放款”
2. **云卡侧能力新增依赖明显增加**：订单状态机、关单逻辑、云卡用信状态查询接口
3. **H5 页面范围扩大**：新增拒绝落地页、签约 loading、行权分发页、协议强制阅读、自动埋点
4. **后端状态控制加强**：未结清不允许再次借款，放弃权益需要通知云卡
5. **法大大虽然暂停开发，但齐为签约成功后触发签章的业务位置已明确**

后续建议先把云卡 / 小花关键接口和状态机补齐，再启动艾博生侧真实改造，避免在关键外部状态未定前过早重构主链路。

---

## 7. 一句话结论

> `2026-04-23` 的核心变化是：用信提交链路从“艾博生经云卡透传小花”升级为“云卡先建放款订单，权益购买和签约完成后再触发放款”，同时新增拒绝落地页、协议强制阅读、行权分发页、未结清限制再借和 H5 自动埋点等一组后续改造事项。
