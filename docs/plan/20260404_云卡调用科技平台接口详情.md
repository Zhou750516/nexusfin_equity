## 2. 接口归属判定

根据 `科技平台接口文档.docx` 的原文定义：

- **机构侧接口定义** = **科技平台提供**，**云卡调用**

- **提供方**：科技平台
- **调用方**：云卡

## 3. 通用接入要求

### 3.1 通信协议

- 协议：`HTTPS`
- 编码：`UTF-8`
- 请求方式：`POST`
- 报文格式：`application/json`
- 要求：接口需保持幂等

### 3.2 公共请求头


| 字段          | 类型       | 必填  | 说明                                  |
| ----------- | -------- | --- | ----------------------------------- |
| `channelId` | `string` | 是   | 平台方 ID，机构方分配给平台的唯一标识；放在 `headers` 中 |


### 3.3 公共请求体


| 字段          | 类型       | 必填  | 说明                |
| ----------- | -------- | --- | ----------------- |
| `requestId` | `string` | 是   | 请求流水，每次请求必须唯一     |
| `timestamp` | `long`   | 是   | 时间戳，毫秒值           |
| `sign`      | `string` | 是   | 签名，`RSA` 非对称加密    |
| `data`      | `string` | 是   | 业务数据密文，`AES` 对称加密 |


### 3.4 公共响应体


| 字段                | 类型              | 必填  | 说明                      |
| ----------------- | --------------- | --- | ----------------------- |
| `code`            | `int`           | 是   | `0` 表示调用成功，非 `0` 表示调用失败 |
| `msg` / `message` | `string`        | 否   | 信息描述                    |
| `timestamp`       | `long`          | 是   | 时间戳                     |
| `data`            | `object/string` | 是   | 业务数据                    |
| `sign`            | `string`        | 是   | 响应签名                    |


### 3.5 加密加签

- 请求和回调都需要先对 `data` 做 `AES` 加密
- 再将加密后的 `data` 与其他参数按约定顺序拼接，做 `RSA` 签名
- 接收方先验签，再解密 `data`
- 响应参数的加密方式与请求保持一致

### 3.6 时效要求

- `2.1 用户准入（撞库）`：响应时间不能超过 **4 秒**
- 其他机构侧接口：响应时间不能超过 **15 秒**

### 3.7 通用返回码


| code    | 含义          |
| ------- | ----------- |
| `0`     | 接口调用成功      |
| `10001` | 内部错误，接口调用失败 |
| `10003` | 参数错误        |
| `10004` | 签名错误        |

## 5. 接口详情

## 5.1 用户准入（撞库）

- **用途**：云卡在推送进件前，先校验科技平台是否接受该用户进件
- **调用说明**：如命中在贷、黑名单、特殊名单等，则不可进件
- **特殊要求**：响应时间不能超过 `4 秒`

### 请求参数


| 字段         | 类型       | 必填  | 说明                         |
| ---------- | -------- | --- | -------------------------- |
| `cidMD5`   | `string` | 是   | 身份证 MD5，32 位小写；加密前 `X` 需大写 |
| `phoneMD5` | `string` | 否   | 手机号 MD5，32 位小写             |


补充说明：

- `cidMD5` 和 `phoneMD5` 至少传一个，也可能两个都传

### 响应字段


| 字段             | 类型       | 必填  | 说明                            |
| -------------- | -------- | --- | ----------------------------- |
| `isCanApply`   | `int`    | 是   | `0-否`，`1-是`                   |
| `rejectReason` | `int`    | 是   | `0-不拒绝`，`1-黑名单`，`2-在贷`，`3-其他` |
| `isStock`      | `int`    | 否   | `0-机构新客`，`1-机构老客`，`3-历史被拒用户`  |
| `remark`       | `string` | 否   | 备注；`rejectReason=3` 时写明具体原因   |


## 5.2 协议列表

- **用途**：获取进件、绑卡、确认借款、放款后查看已签署协议等环节的协议列表

### 请求参数


| 字段            | 类型       | 必填  | 说明                                                          |
| ------------- | -------- | --- | ----------------------------------------------------------- |
| `userId`      | `string` | 是   | 用户唯一标识；`optType=1` 时不传                                      |
| `applyId`     | `int`    | 是   | `optType=1` 时不传；`optType=2/3` 传平台授信流水号；`optType=4` 传平台借款流水号 |
| `optType`     | `int`    | 是   | `1-进件申请`，`2-确认借款`，`3-绑卡`，`4-放款后查看已签署协议`                     |
| `loanAmount`  | `double` | 否   | `optType=2` 时传，单位元                                          |
| `loanPeriod`  | `int`    | 否   | `optType=2` 时传，单位月                                          |
| `loanReason`  | `string` | 否   | `optType=2` 时传，借款用途码表                                       |
| `name`        | `string` | 否   | `optType=1` 时可传                                             |
| `cid`         | `string` | 否   | `optType=1` 时可传                                             |
| `bankCardNum` | `string` | 否   | `optType=2/3` 时可传                                           |
| `phone`       | `string` | 否   | `optType=2/3` 时可传                                           |


### 响应字段


| 字段     | 类型      | 必填  | 说明   |
| ------ | ------- | --- | ---- |
| `list` | `array` | 是   | 协议列表 |


协议列表元素：


| 字段       | 类型       | 必填  | 说明            |
| -------- | -------- | --- | ------------- |
| `title`  | `string` | 是   | 合同名称          |
| `isShow` | `int`    | 是   | `1-强展示`，`2-否` |
| `url`    | `string` | 是   | 协议链接          |


## 5.3 进件推送

- **用途**：云卡向科技平台推送授信 / 进件申请信息
- **备注**：
  - 图片类信息不在本接口中同步
  - 科技平台接收进件文本后，再通过云卡提供的图片资源拉取接口获取身份证照片、活体照片
  - 收到请求后可先返回进件成功，拉取照片和审核流程异步处理，审核结果通过通知接口推送

### 请求参数（顶层）


| 字段               | 类型       | 必填  | 说明      |
| ---------------- | -------- | --- | ------- |
| `userId`         | `string` | 是   | 用户唯一标识  |
| `applyId`        | `int`    | 是   | 平台授信流水号 |
| `basicInfo`      | `object` | 是   | 基本信息    |
| `idInfo`         | `object` | 是   | 实名信息    |
| `contactInfo`    | `array`  | 是   | 紧急联系人信息 |
| `supplementInfo` | `object` | 是   | 补充信息    |
| `deviceInfo`     | `object` | 是   | 设备信息    |
| `agentId`        | `int`    | 是   | 用户渠道 ID |
| `agentName`      | `string` | 是   | 用户渠道名称  |
| `extendInfo`     | `object` | 否   | 扩展字段    |


### 关键业务字段

- `basicInfo`：手机号、住宅地址、公司信息、学历、婚姻、月收入
- `idInfo`：姓名、身份证号、性别、身份证住址、签发机关、有效期、民族
- `contactInfo`：联系人姓名、电话、关系、顺序
- `supplementInfo`：职业
- `deviceInfo`：设备标识、系统信息、网络信息、IP、是否越狱 / root 等

### 响应字段


| 字段        | 类型       | 必填  | 说明                           |
| --------- | -------- | --- | ---------------------------- |
| `status`  | `int`    | 是   | `2001-进件成功`，`2002-进件失败`      |
| `isStock` | `int`    | 否   | `0-机构新客`，`1-机构老客`，`3-历史被拒用户` |
| `remark`  | `string` | 否   | 失败原因描述                       |


## 5.4 审核结果查询

- **用途**：在审核中时，由云卡主动查询科技平台审核 / 授信结果
- **备注**：授信成功或失败时，科技平台还需通过回调接口通知云卡

### 请求参数


| 字段        | 类型       | 必填  | 说明      |
| --------- | -------- | --- | ------- |
| `userId`  | `string` | 是   | 用户唯一标识  |
| `applyId` | `int`    | 是   | 平台授信流水号 |


### 响应字段


| 字段                   | 类型       | 必填  | 说明                                               |
| -------------------- | -------- | --- | ------------------------------------------------ |
| `status`             | `int`    | 是   | `6002-审核通过`，`6003-审核处理中`，`6004-审核拒绝`，`6008-贷款取消` |
| `rejectReason`       | `int`    | 否   | 审核拒绝原因码                                          |
| `remark`             | `string` | 否   | 描述                                               |
| `amountOption`       | `array`  | 否   | 审核通过时必传，可选金额周期                                   |
| `totalAmount`        | `double` | 否   | 授信总金额，审核通过时必传                                    |
| `surplusAmount`      | `double` | 否   | 剩余可用金额，审核通过时必传                                   |
| `yearRate`           | `double` | 否   | 年利率，审核通过时必传                                      |
| `creditEndTime`      | `long`   | 否   | 额度失效时间                                           |
| `creditUnfrozenTime` | `long`   | 否   | 额度冻结结束时间                                         |
| `isStock`            | `int`    | 否   | 新老客标识                                            |
| `fundInfo`           | `object` | 否   | 资金提供方信息                                          |


补充说明：

- `amountOption.periods` 表示可选借款周期，如 `[3,6,9,12]`
- `fundInfo` 包含 `fundId`、`fundName`
