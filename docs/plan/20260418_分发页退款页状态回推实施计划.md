# 分发页 / 退款页 / 状态回推实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在联合登录核心入口骨架已落地的基础上，把 `/joint-dispatch`、`/joint-refund-entry` 和权益状态回推能力推进到可分阶段实施、可联调验收的状态。

**Architecture:** 方案分三条主线推进：第一条是“分发页上下文查询 + 跳转解析”，第二条是“退款入口信息查询 + 退款申请 / 结果查询”，第三条是“权益状态事件组装 + 主动回推科技平台”。三条主线统一复用联合登录后的本地身份态、`benefitOrderNo` 上下文和现有 `BenefitOrder` / `MemberInfo` 数据模型，并尽量把外部协议变化封装在后端服务与客户端层。

**Tech Stack:** Java 17, Spring Boot 3.2, MockMvc, MyBatis-Plus, React 19, TypeScript 5.6, axios, wouter, Vitest

---

## Scope Decision

本计划覆盖三块后续正式实施入口：

1. 分发页正式承接能力
2. 退款页正式承接能力
3. 权益状态回推基础能力

本计划不包含：

1. 小花真实联合登录协议实现
2. 齐为真实供应商 URL 获取细节
3. 齐为真实退款协议字段最终确定
4. 科技平台状态回推最终签名细节

这些外部契约由已有方案文档继续承接，实施时按最新外部文档替换客户端适配层即可。

```ts
export function resolveDispatchPageState(input: {
  allowRedirect: boolean;
  redirectMode: "DIRECT" | "INTERMEDIATE";
  supplierUrl?: string;
}) {
  if (input.allowRedirect && input.redirectMode === "DIRECT" && input.supplierUrl) {
    return { type: "redirect", url: input.supplierUrl } as const;
  }
  return { type: "info" } as const;
}
```
