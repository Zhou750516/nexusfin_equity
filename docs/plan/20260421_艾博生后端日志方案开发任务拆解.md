# 2026-04-21 艾博生后端日志方案开发任务拆解

## 1. 任务目标

基于 `plan/20260421_艾博生后端日志完备性Review计划.md:1` 已确认的日志方案，进一步拆解为可执行的开发任务。

本轮开发任务目标：

1. 统一艾博生后端日志等级口径；
2. 统一错误日志的 `errorNo / errorMsg` 输出；
3. 打通“接口入口 → Service → 云卡调用 → 返回 / 异常 / 补偿”的 `traceId` 串联；
4. 补齐云卡调用前后日志闭环；
5. 补齐统一异常日志；
6. 补齐最小化验证与联调检查项。

## 2. 开发范围

本轮任务只覆盖当前主链路相关模块：

- 借款链路
- 还款链路
- 云卡网关调用链路
- 统一异常处理
- 配置层日志等级开关
- 必要的最小化测试 / 验证材料

暂不扩展到：

- 全量历史模块的日志重构
- 监控平台 / SLS / 告警规则接入
- 所有第三方客户端的统一改造

## 3. 开发原则

### 3.1 日志等级

统一按以下四级实施：

- `DEBUG`
- `INFO`
- `WARN`
- `ERROR`

### 3.2 关键字段

关键日志字段至少包括：

- `traceId`
- `bizOrderNo`
- `requestId`
- `path`
- `errorNo`
- `errorMsg`

按场景补充：

- `memberId`
- `applicationId`
- `loanId`
- `repaymentId`
- `taskType`
- `workerId`

### 3.3 安全要求

日志中禁止直接输出以下敏感明文：

- 身份证号
- 手机号完整值
- 银行卡完整卡号
- token 完整值
- 签名密钥
- 完整密文 / 完整验签串

如需输出，只能输出：

- 脱敏值
- hash
- 长度
- 业务编号
- 路径
- 请求编号

### Task 3：补云卡客户端出站 begin / success / fail 日志

**目标**

让每次请求云卡都具备完整日志闭环，并能用 `traceId` 串起来。

**涉及范围**

- `src/main/java/com/nexusfin/equity/thirdparty/yunka/RestYunkaGatewayClient.java`

**建议改动**

1. 调用前打印 `begin` 日志；
2. 返回成功后打印 `success` 日志；
3. 超时打印 `error` 日志；
4. 非超时异常打印 `error` 日志；
5. 上游业务失败 / 空响应按语义打印 `warn` 或 `error`；
6. 记录耗时。

**建议日志字段**

- `traceId`
- `bizOrderNo`
- `requestId`
- `path`
- `elapsedMs`
- `errorNo`
- `errorMsg`

**完成标准**

- 任意一次云卡调用都能查到 begin / success / fail；
- 同一调用链中的 `traceId` 一致；
- 能区分超时、异常、空响应、业务失败。

**优先级**

- P0

### Task 4：补借款链路 Service 日志闭环

**目标**

补齐借款主链路中“调用云卡前后”和“关键业务分支”的日志。

**涉及范围**

- `src/main/java/com/nexusfin/equity/service/impl/LoanServiceImpl.java`

**重点场景**

- `calculate`
- `apply`
- `approval-status`
- `approval-result`

**建议改动**

1. 调云卡前打印业务 begin 日志；
2. 调云卡成功后打印 success 日志；
3. 返回前端 `pending` / `loan_failed` / `approved` / `rejected` 时打印结果日志；
4. 超时转异步补偿时打印 `warn` 日志；
5. 权益购买成功但借款失败的分支要有明确日志。

**完成标准**

- 借款链路的关键分支有清晰日志；
- `loan apply` 能串起：
  - 入口
  - 购买权益
  - 调云卡
  - 超时 / 失败 / 成功
  - 返回前端口径

**优先级**

- P0
