# 2026-04-12 H5 Phase 9 任务拆解

基于已确认的实施决策，按“先简单、后转发、再编排”的顺序展开 `Phase 9` 执行任务。

## 1.1 当前执行前提

1. 云卡服务当前由其他同学并行开发。
2. 艾博生侧现阶段按 `20260410_艾博生调用云卡接口文档.md` 完成统一 gateway 接入封装、DTO 映射和调用链路实现。
3. 当前任务可先完成本地开发、自测与对接准备，待云卡服务环境可用后，再按既定口径开展真实联调。

## 2. 任务分组

### 任务组 A：本地配置与齐为直连能力

**目标**

- 先打通不依赖云卡 gateway 的接口，尽快完成最小可联调闭环。

**任务**

1. 实现 `GET /api/loan/calculator-config`
   - 使用本地配置
   - 固定 3 期
2. 实现 `GET /api/benefits/card-detail`
   - 适配齐为权益信息接口
3. 实现 `POST /api/benefits/activate`
   - 适配齐为权益开通接口
4. 完成对应 DTO、映射与测试

**完成标准**

1. H5 首页与权益页可以不依赖云卡先完成真实数据联调
2. 3 个接口返回结构符合 `H5/API_SCHEMA.md`

### 任务组 B：云卡 gateway 基础设施

**目标**

- 沉淀统一的云卡请求封装和响应解包能力。

**任务**

1. 新增云卡 gateway 客户端
2. 支持统一 gateway URI
3. 支持统一请求结构：
   - `requestId`
   - `path`
   - `bizOrderNo`
   - `data`
4. 将 `/loan/trail` 路径抽成配置项，默认值 `/loan/trail`
5. 补充转发请求封装测试

**完成标准**

1. 后续所有云卡转发接口均复用同一客户端
2. 路径变更仅需调整配置
3. 在云卡服务未就绪前，可基于接口文档完成封装与联调前准备

### 任务组 C：贷款查询类接口

**目标**

- 打通贷款试算和审批查询类接口。

**任务**

1. 实现 `POST /api/loan/calculate`
   - 转发 `/loan/trail`
2. 实现 `GET /api/loan/approval-status/{applicationId}`
   - 根据本地 `applicationId` 映射恢复上游查询参数
   - 转发 `/loan/query`
   - 映射为“审批中”页面 DTO
3. 实现 `GET /api/loan/approval-result/{applicationId}`
   - 根据本地 `applicationId` 映射恢复上游查询参数
   - 转发 `/loan/query`
   - 映射为“审批结果”页面 DTO
4. 完成对应 DTO 和测试

**完成标准**

1. 两个查询接口共用上游 `/loan/query`
2. 但对 H5 返回两套不同页面视图模型

### 任务组 D：还款链路接口

**目标**

- 打通还款详情、提交和结果查询链路。

**任务**

1. 实现 `GET /api/repayment/info/{loanId}`
   - 转发 `/repay/trial`
2. 实现 `POST /api/repayment/submit`
   - 转发 `/repay/apply`
3. 实现 `GET /api/repayment/result/{repaymentId}`
   - 使用科技平台跳转参数中的 `repaymentId`
   - 转发 `/repay/query`
4. 完成对应 DTO 和测试

**完成标准**

1. 不为 `repaymentId` 额外生成本地编号
2. 还款结果接口按既有跳转参数和上下文完成查询

### 任务组 E：借款申请编排接口

**目标**

- 打通最复杂的复合编排接口 `POST /api/loan/apply`。

**任务**

1. 本地生成 `applicationId`
2. 保存 `applicationId` 与上游查询参数映射
3. 先调用齐为购买权益
4. 权益成功后再调用云卡 `/loan/apply`
5. 处理部分成功场景：
   - 若齐为成功、云卡失败
   - 返回借款失败
   - 明确提示权益购买成功
6. 补充编排顺序测试和部分成功测试

**完成标准**

1. 满足“先齐为、后云卡”
2. 部分成功场景状态可区分、可追踪

## 3. 推荐执行顺序

## 4. 执行前置条件

开始正式实现前，以下条件已视为完成：

1. `/loan/trail` 使用配置项，默认值已确认
2. `approval-status` / `approval-result` 共用 `/loan/query`
3. `applicationId` 允许本地生成并维护映射
4. `repaymentId` 使用科技平台跳转参数
5. 齐为成功、云卡失败时返回借款失败，但保留“权益购买成功”语义
