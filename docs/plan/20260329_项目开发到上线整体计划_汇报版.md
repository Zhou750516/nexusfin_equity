# 惠聚项目开发到上线整体计划（汇报版）

文档日期：2026-03-29  
汇报版更新：2026-03-31

## 1. 项目当前结论

截至 `2026-03-31`，项目整体判断为：

**开发主链已完成，联调准备内部项已基本完成，当前主要阻塞在外部协同条件确认，项目正处于从 M2 向 M3 过渡阶段。**

当前已完成的核心工作：

1. 权益服务基线能力已落地，包含订单创建、回调处理、幂等、审计与敏感信息加密。
2. 跨域认证主链已落地，包含 `SSO` 回调登录、本地 `JWT + Cookie`、`/api/users/me` 与业务接口认证边界。
3. 齐为协议适配层与 3 个主动调用接口已进入代码主链。
4. 当前代码已完成 `mvn test`、MySQL 回归、`package`、`checkstyle` 验证。
5. 联调准备文档已形成，包含联调前检查清单、待外部确认项清单、出站接口联调执行说明。

需要单独说明的是：

**当前汇报主线聚焦“齐为联调与上线交付主线”；云卡完整页面交互与借还款编排属于独立开发主线，应单独排期、单独汇报，不与当前联调结论混写。**

## 3. 当前重点阻塞项

| 阻塞项 | 当前缺口 | 对项目的直接影响 | 优先级 |
| --- | --- | --- | --- |
| 齐为真实联调参数 | `partnerNo`、`signKey`、`aesKeyBase64`、测试网关地址未最终确认 | `M3.1` 无法启动真实出站联调 | 高 |
| 科技平台真实测试环境 | `/api/users/me` 真实地址、Bearer Token 样例、返回字段结构待确认 | `SSO` 真实链路无法闭环，登录态验收无法关闭 | 高 |
| 扣款回调映射规则 | 首扣与兜底代扣区分方式未最终明确 | `M3.2` 入站回调联调无法稳定排期 | 高 |
| 公网测试回调地址与白名单 | 测试域名、暴露路径、白名单策略待确认 | 齐为无法真实回调我方测试环境 | 高 |
| 验收与上线文档收口 | 上线说明、最终验收记录、遗留问题归档尚未完整形成 | 会压缩 `M4` 与 `M5` 时间窗口 | 中 |
