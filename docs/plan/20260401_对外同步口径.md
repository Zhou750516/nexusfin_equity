# 2026-04-01 对外同步口径

## 1. 使用说明

本文档用于在 `2026-04-01` 当天向外部协同方同步当前联调推进状态，目标是：

1. 让对方快速看到“当前缺什么”
2. 让对方明确“不给什么会卡住哪个里程碑”
3. 避免沟通停留在“还差点信息”这种模糊表达

## 2. 发给齐为

建议同步口径如下：

> 目前我方本地开发主链已完成，正在推进 `M2 -> M3.1` 联调主线。  
> 为尽快启动真实环境联调，请协助今天确认以下信息：  
> 1. 测试环境最终网关地址、`methodPath`、协议版本口径  
> 2. 联调所需正式测试参数：`partnerNo`、`signKey`、`aesKeyBase64`  
> 3. `abs.push.order`、`abs.token.get`、`abs.lending.notify` 三个接口的成功 / 失败样例  
> 4. 扣款通知如何区分首扣与兜底代扣，请提供字段定义、判定规则和样例报文  
> 5. 行权、退款、还款回调的最小样例报文  
> 当前若以上信息未齐，`M3.1` 无法进入真实 `HTTP` 联调，`M3.2` 的入站回调联调也无法稳定排期。

## 3. 发给科技平台

建议同步口径如下：

> 目前权益服务已完成本地主链验证，正在收口 `M2` 外部条件。  
> 为完成真实环境闭环验证，请协助今天确认以下信息：  
> 1. 测试环境 `baseUrl` 与 `/api/users/me` 真实地址  
> 2. Bearer Token 获取方式，或可直接使用的测试 token 样例  
> 3. `/api/users/me` 真实返回字段结构与样例  
> 当前若以上信息未齐，SSO 真联调链路无法闭环，`M2` 也无法从“内部准备完成”推进到“外部条件确认完成”。

## 4. 发给我方网关 / 运维

建议同步口径如下：

> 当前项目正在推进齐为联调主线，需要尽快确认测试环境回调暴露条件。  
> 请协助确认：  
> 1. 可供齐为访问的公网测试域名  
> 2. 回调路径的对外暴露方式  
> 3. 是否需要来源 IP 或域名白名单，以及具体配置方式  
> 当前若以上信息未齐，齐为无法回调我方测试环境，`M3.2` 入站联调无法排期启动。

## 5. 对内统一状态口径

建议当天统一使用以下描述：

1. 当前代码主链已基本就绪，阻塞不在本地实现。
2. 当前最关键阻塞在外部确认项，而不是继续泛化开发。
3. 若当天拿齐关键参数与地址，项目可直接从 `M2` 切入 `M3.1`。
4. 若当天仍拿不到关键条件，当天产出应聚焦于阻塞固化、责任方催办视图和时间线判断。
