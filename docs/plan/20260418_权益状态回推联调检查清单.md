# 2026-04-18 权益状态回推联调检查清单

## 1. 适用范围

本文用于联调权益状态回推当前已落地的最小骨架，验证范围包括：

1. `BenefitStatusPushService`
2. `BenefitStatusPushLog`
3. `BenefitStatusPushLogRepository`
4. `SkeletonTechPlatformBenefitStatusClient`

---

## 2. 联调前检查

### 后端

1. 确认后端代码已包含 `benefit_status_push_log` 表结构
2. 确认 `BenefitStatusPushService` 与 `SkeletonTechPlatformBenefitStatusClient` 已注册
3. 确认测试环境可运行聚焦单测

---

## 3. 手工执行步骤

### 场景 A：状态回推服务触发

1. 调用：

```text
BenefitStatusPushService.pushStatus(...)
```

2. 检查首次写入的日志字段：
   - `eventId`
   - `benefitOrderNo`
   - `eventType`
   - `pushStatus=INIT`

### 场景 B：回推成功后更新日志

1. 当外部客户端返回成功
2. 检查后续更新日志字段：
   - `pushStatus=SUCCESS`
   - `responsePayload`

### 场景 C：Skeleton 客户端占位

1. 当使用 `SkeletonTechPlatformBenefitStatusClient`
2. 检查是否明确暴露：

```text
TECH_PLATFORM_STATUS_PUSH_NOT_READY
```

---

## 4. 关键检查项

1. 日志表是否已在主库 schema 与 H2 schema 中补齐
2. `eventId` 是否作为主键使用
3. 状态回推日志是否能区分：
   - 初始写入
   - 推送成功
   - 后续失败扩展位
4. 服务日志是否带：
   - `traceId`
   - `bizOrderNo`

---

## 5. 当前说明

本轮仅验证“权益状态回推骨架”。

当前仍未包含：

1. 科技平台真实 HTTP 回推
2. 幂等与重试调度
3. 回推失败补偿机制
