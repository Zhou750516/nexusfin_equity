# 2026-04-15 发小花联合登录待确认清单

## 1. 目的

本文档用于整理一版可直接发给小花科技同学的“联合登录待确认问题清单”，便于对方补齐调用文档，减少后续来回沟通成本。

---

## 2. 背景口径

当前双方已确认的联合登录方案为：

1. 科技平台跳转到艾博生页面时，携带 `token`
2. 艾博生页面请求艾博生后端
3. 艾博生后端再调用小花后端完成用户身份换取 / 校验 / 上下文建立

基于该方案，为便于艾博生后续调用与落地，请小花侧协助补齐以下资料。

---

## 3. 待确认问题清单

### 3.1 跳转入参

1. 科技平台跳转到艾博生页面时，固定会携带哪些参数？
2. `token` 的参数名是什么？
3. 除 `token` 外，是否还会携带：
   - 用户标识
   - 渠道标识
   - 订单号
   - 场景标识
   - 时间戳 / nonce / 签名
4. 跳转 URL 是否有固定格式示例？

### 3.2 token 口径

1. `token` 的有效期是多少？
2. `token` 是否一次性使用？
3. `token` 是否需要签名校验或加密解密？
4. `token` 失效后，是否支持重新换取？
5. `token` 校验失败时，标准错误码和错误文案是什么？

### 3.3 艾博生后端调用小花后端的接口

1. 艾博生后端调用小花后端时，请提供正式接口文档：
   - 请求地址
   - 请求方式
   - 请求头
   - 请求体
   - 返回体
2. 是否存在沙箱 / 测试环境地址？
3. 是否需要白名单？
4. 是否需要签名、时间戳、防重放机制？
5. 调用超时、重试、幂等是否有要求？

### 3.4 用户身份换取结果

1. 联合登录成功后，小花侧返回的用户唯一标识字段是什么？
2. 该用户标识是否稳定不变？
3. 后续业务接口是否统一使用该用户标识？
4. 是否会返回用户基础信息（如手机号、实名状态、渠道信息）？

### 3.5 失败场景

1. 如果 `token` 无效，前端 / 后端应如何处理？
2. 如果小花后端接口调用失败，是否有标准降级口径？
3. 如果用户状态异常（冻结、未实名、无权限），是否有固定返回码？
4. 是否需要前端区分“可重试”和“不可重试”的错误场景？

### 3.6 联调支持

1. 是否可提供联合登录时序图？
2. 是否可提供一组测试环境样例参数？
3. 是否可提供联调用的 mock 用户 / token 样例？
4. 是否有推荐的联调顺序或注意事项？
