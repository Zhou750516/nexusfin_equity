# 异步消息补偿机制开发任务拆解 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 基于本地 MySQL 落地方案 A 的异步消息补偿机制，支持齐为权益购买失败补偿、云卡/小花用信提交超时补偿，以及 partition worker + supervisor 常驻治理能力。

**Architecture:** 新增独立补偿任务主表、尝试日志表和分片运行表，把“业务状态”与“调度状态”分离。同步链路只负责识别超时/失败、落本地任务并返回对应前端口径；异步链路通过分片 worker 串行消费任务，supervisor 负责心跳、积压、租约回收和告警。

**Tech Stack:** Java 17, Spring Boot 3.2, MyBatis-Plus, MySQL 8.0, H2, MockMvc, Mockito, Jackson

---

## Scope Decision

本计划只覆盖方案 A 的第一阶段可上线实现，不引入外部 MQ，不做动态 rebalance。

**本计划包含：**

1. MySQL 持久化补偿任务模型
2. 齐为权益购买失败入队
3. 云卡用信提交超时入队，并返回前端“已提交 / 审核中”
4. partition worker 消费链路
5. supervisor 心跳 / 租约回收 / 积压检查
6. 单元测试、集成测试、联调 checklist

**本计划不包含：**

1. 外部 MQ 替换
2. supervisor 自动拉起 OS 进程
3. 动态分片扩缩容
4. 所有下游场景统一接入补偿框架

```java
public void pollAndExecute(String workerId, int partitionNo) {
    AsyncCompensationTask task = taskRepository.selectOne(
            Wrappers.<AsyncCompensationTask>lambdaQuery()
                    .eq(AsyncCompensationTask::getPartitionNo, partitionNo)
                    .in(AsyncCompensationTask::getTaskStatus, "INIT", "RETRY_WAIT")
                    .le(AsyncCompensationTask::getNextRetryTs, LocalDateTime.now())
                    .orderByAsc(AsyncCompensationTask::getCreatedTs)
                    .last("limit 1")
    );
    if (task == null) {
        return;
    }
    task.setTaskStatus("PROCESSING");
    task.setLeaseOwner(workerId);
    task.setLeaseExpireTs(LocalDateTime.now().plusSeconds(properties.getLeaseSeconds()));
    task.setUpdatedTs(LocalDateTime.now());
    taskRepository.updateById(task);
    try {
        routerExecutor.execute(task);
        task.setTaskStatus("SUCCESS");
        task.setLeaseOwner(null);
        task.setLeaseExpireTs(null);
        task.setSuccessTs(LocalDateTime.now());
        task.setUpdatedTs(LocalDateTime.now());
        taskRepository.updateById(task);
    } catch (BizException | UpstreamTimeoutException exception) {
        task.setRetryCount(task.getRetryCount() + 1);
        task.setLeaseOwner(null);
        task.setLeaseExpireTs(null);
        task.setLastErrorMessage(exception.getMessage());
        task.setUpdatedTs(LocalDateTime.now());
        if (task.getRetryCount() >= task.getMaxRetryCount()) {
            task.setTaskStatus("DEAD");
        } else {
            task.setTaskStatus("RETRY_WAIT");
            task.setNextRetryTs(LocalDateTime.now().plusMinutes(1));
        }
        taskRepository.updateById(task);
    }
}
```

```java
public void recycleExpiredLeases() {
    List<AsyncCompensationTask> expiredTasks = taskRepository.selectList(
            Wrappers.<AsyncCompensationTask>lambdaQuery()
                    .eq(AsyncCompensationTask::getTaskStatus, "PROCESSING")
                    .lt(AsyncCompensationTask::getLeaseExpireTs, LocalDateTime.now())
    );
    for (AsyncCompensationTask task : expiredTasks) {
        task.setTaskStatus(AsyncCompensationTaskStatusEnum.RETRY_WAIT.name());
        task.setLeaseOwner(null);
        task.setLeaseExpireTs(null);
        task.setNextRetryTs(LocalDateTime.now().plusMinutes(1));
        task.setUpdatedTs(LocalDateTime.now());
        taskRepository.updateById(task);
    }
}
```
