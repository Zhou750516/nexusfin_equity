# 2026-04-17 小花联合登录联调检查清单

## 1. 适用范围

本文用于联调“小花联合登录核心入口（本地骨架）”的最小闭环，验证范围包括：

1. H5 `/joint-entry` 承接外部跳转参数
2. 后端 `/api/auth/joint-login` 建立本地 `NEXUSFIN_AUTH` Cookie
3. H5 根据返回的 `targetPage` 完成场景分流
4. 刷新后仍可通过 `/api/users/me` 获取当前用户

---

## 2. 联调前检查

### 后端

1. 确认后端已启动
2. 确认 `/api/auth/joint-login` 可访问
3. 确认 `/api/users/me` 仍由 `NEXUSFIN_AUTH` Cookie 鉴权

### H5

1. 确认 H5 本地开发环境已启动
2. 确认 `/api/*` 代理仍指向本地 Java 后端
3. 确认浏览器未残留异常旧 Cookie，可按需清理

---

## 3. 手工执行步骤

### 场景 A：`push` 分发页占位

1. 打开：

```text
/joint-entry?token=joint-token-001&scene=push&benefitOrderNo=BEN-001
```

2. 检查页面是否先进入“联合登录中”加载态
3. 检查是否收到 `NEXUSFIN_AUTH` Cookie
4. 检查页面是否自动跳转到：

```text
/joint-dispatch?scene=push&benefitOrderNo=BEN-001
```

5. 检查分发页是否展示：
   - 联合登录成功后的占位标题
   - `benefitOrderNo`
   - 后续说明文案

### 场景 B：`refund` 退款入口占位

1. 打开：

```text
/joint-entry?token=joint-token-002&scene=refund&benefitOrderNo=BEN-002
```

2. 检查是否自动跳转到：

```text
/joint-refund-entry?scene=refund&benefitOrderNo=BEN-002
```

3. 检查退款入口页是否展示：
   - 占位标题
   - `benefitOrderNo`
   - 当前说明文案

### 场景 C：缺失参数

1. 打开：

```text
/joint-entry?scene=push
```

2. 检查是否进入错误提示态
3. 检查错误文案是否说明“缺少联合登录参数”

---

## 4. 关键检查项

### 登录态

1. `NEXUSFIN_AUTH` Cookie 是否创建成功
2. 刷新目标页后 Cookie 是否仍可用
3. 访问 `/api/users/me` 是否返回当前用户

### 场景路由

1. `push` 是否进入 `/joint-dispatch`
2. `refund` 是否进入 `/joint-refund-entry`
3. 非支持场景是否进入兜底页 / 404

### 页面表现

1. 入口页是否有加载态
2. 参数错误时是否有错误态
3. 新页面是否支持现有多语言框架

---

## 5. 当前说明

本轮只验证“联合登录核心入口骨架”。

当前仍未包含：

1. 小花真实联调协议
2. 齐为真实供应商页面跳转
3. 正式退款申请 / 退款结果页面
4. 权益状态回推科技平台

上述内容需要在外部正式文档补齐后继续扩展。
