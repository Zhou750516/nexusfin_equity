# 2026-04-21 小花科技会议纪要与行动计划

## 1. 文档目标

本文档用于沉淀 `2026-04-21` 与小花科技的最新沟通结论，并将新增确认项、待确认项和我方行动项统一收口，作为后续联合登录、前后端报文加密和 `partnerId` 方案推进的基线文档。

本次会议重点包括：

1. 艾博生后端调用小花后端登录验证接口
2. 小花未登录状态下页面访问与弹窗登录能力
3. 艾博生 H5 调后端接口的报文加密要求
4. `partnerId` 由艾博生生成
5. 小花补文档并同步排期

---

## 2. 会议结论

### 2.1 联合登录第一步：艾博生后端调用小花登录验证接口

本次会议确认：

1. 艾博生后端调用小花科技后端的登录验证接口
2. 该接口用于判断用户是否已登录
3. 该接口返回用户的登录信息

当前含义：

1. 联合登录主链路继续保持“艾博生前端 -> 艾博生后端 -> 小花后端”
2. 小花登录验证接口是联合登录第一阶段必须明确的真实外部接口
3. 艾博生后续需要基于该接口返回结果建立本地登录态和用户上下文

### 2.2 小花内部确认未登录状态访问能力

本次会议确认：

1. 小花内部还需确认，是否支持用户在未登录状态下访问小花页面
2. 若未登录，页面是否可以直接弹出登录页

当前含义：

1. 该事项会影响艾博生跳转小花页面时的兜底体验
2. 若小花支持未登录访问后弹登录，则跳转链路容错更高
3. 若小花不支持，则艾博生侧需要更严格地保证跳转前已完成身份校验

当前状态：

- `@待小花同学确认`

### 2.3 艾博生前端调用后端接口需要报文加密

本次会议确认：

1. 艾博生的前端页面访问艾博生后端接口时，需要做报文加密

当前含义：

1. 报文加密不再只是“小花接口接入”的外部加密问题
2. 还需要覆盖“艾博生 H5 -> 艾博生后端”的内部接口调用链路
3. 后续需要单独明确：
   - 加密范围是全部接口还是指定敏感接口
   - 加密算法、密钥管理、签名 / 验签、时间戳 / nonce 规则
   - 前端和后端各自的职责边界

### 2.4 `partnerId` 由艾博生生成

本次会议确认：

1. `partnerId` 由艾博生生成
2. 后续艾博生调用小花科技接口时，使用该 `partnerId`

当前含义：

1. `partnerId` 来源方案已从“待小花确认”收口为“艾博生生成”
2. 艾博生后续需要明确 `partnerId` 生成规则、唯一性策略、落库 / 映射策略和传递口径
3. 联合登录、用户映射、后续调用模型都需要按该结论回填

### 2.5 小花同步文档并给出排期计划

本次会议确认：

1. 小花科技后续同步正式文档
2. 小花科技给出排期计划

当前含义：

1. 小花侧正式接口文档与排期计划，仍是后续真实协议接入的关键外部前提
2. 艾博生侧当前仍可先做方案骨架和字段映射准备
3. 在文档未到位前，不建议直接进入真实协议开发

## 4. 行动计划

### P0：回填 `partnerId` 新结论

**目标**

- 将 `partnerId` 的来源口径统一切换为“艾博生生成”

**行动项**

1. 更新小花相关待办清单中的 `partnerId` 状态
2. 更新联合登录相关方案文档中 `partnerId` 的来源描述
3. 后续补充：
   - `partnerId` 生成规则
   - 唯一性与幂等口径
   - 本地落库 / 映射策略
   - 传给小花的入参位置

### P0：联合登录真实接口继续收口

**目标**

- 以“小花登录验证接口”为真实联合登录接入的第一优先级

**行动项**

1. 等小花补齐登录验证接口文档
2. 明确接口的请求方式、请求字段、响应字段、错误码
3. 明确返回用户登录信息的字段口径
4. 基于该接口继续更新联合登录真实协议适配文档

### P0：梳理艾博生 H5 -> 后端报文加密方案

**目标**

- 把前端调后端接口的报文加密从会议结论转化为可执行方案

**行动项**

1. 确认加密范围：
   - 全量接口
   - 仅敏感接口
2. 确认前端职责：
   - 加密
   - 签名
   - 时间戳 / nonce
3. 确认后端职责：
   - 解密
   - 验签
   - 防重放
4. 形成独立技术方案或并入联合登录 / H5 安全方案文档

### P1：跟进小花未登录访问能力

**目标**

- 明确艾博生跳转小花页面时，是否需要额外做登录兜底控制

**行动项**

1. 等小花确认未登录访问与弹窗登录能力
2. 若支持，则记录为页面跳转兜底能力
3. 若不支持，则回填到艾博生跳转前置校验方案

### P1：跟进小花文档与排期

**目标**

- 保持真实协议推进有明确外部时间点

**行动项**

1. 等小花同步最新文档
2. 等小花给出排期计划
3. 文档与排期到位后，再拆真实开发和联调计划
