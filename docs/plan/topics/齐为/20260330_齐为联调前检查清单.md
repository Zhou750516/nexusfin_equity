## 6. 成功 / 失败 / 异常场景验证项

### 6.1 成功场景

1. 下单后成功调用 `abs.push.order`，返回齐为订单号和卡号。
2. 查询行权链接成功返回 `redirectUrl`。
3. 放款成功通知齐为后返回成功响应。
4. 扣款、行权、退款、还款回调都能被本地正确验签、落日志、更新状态。

### 6.2 失败场景

1. 齐为返回业务失败码时，我方能正确抛错并阻断后续误更新。
2. 回调签名错误时，我方拒绝请求。
3. 回调关联订单不存在时，我方记录 `FAILED` 审计状态而不是误记成功。

### 6.3 异常场景

1. 齐为接口超时或网络失败时，我方返回明确错误。
2. 齐为响应密文解密失败时，我方能识别为协议异常。
3. 同一请求重复回放时，我方幂等逻辑不重复落单或重复处理回调。
