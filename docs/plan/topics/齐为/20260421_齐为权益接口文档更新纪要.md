## 2. 本次新增内容

### 2.1 会员同步接口新增字段

齐为文档本次明确：会员同步接口新增以下字段：

1. `mobile`
2. `username`
3. `accountNo`
4. `cardNo`

当前含义：

1. 艾博生侧现有会员同步请求模型后续需要补字段映射评估
2. 需要确认这些字段是全部必填，还是按场景选填
3. 需要确认字段来源是否都能在本地直接取到
4. 需要特别确认：
   - `accountNo` 与 `cardNo` 是否为不同业务含义
   - 敏感字段是否涉及加密、脱敏和落库限制

### 2.2 新增 3 个签约相关接口

齐为文档本次明确：新增以下 3 个接口：

1. 签约查询
2. 签约申请
3. 签约确认

这 3 个接口与此前已记录的绑卡 TODO 可对应为：

| 之前的记录口径 | 当前齐为文档口径 |
|---|---|
| 查看银行卡是否已签约 | 签约查询 |
| 申请绑卡（发短信） | 签约申请 |
| 绑卡确认（验短信） | 签约确认 |

当前含义：

1. 齐为绑卡方向已从“纯需求描述”推进到“接口已进入文档”
2. 艾博生侧后续可以开始做接口承接映射设计
3. 但在字段、状态、错误码和时序未完全确认前，仍不建议直接进入真实开发

## 3. 对艾博生侧的影响

### 3.1 会员同步方向

1. 现有会员同步请求模型是否需要扩字段
2. 本地是否已有以下字段来源：
   - `mobile`
   - `username`
   - `accountNo`
   - `cardNo`
3. 若本地无直接字段来源，需明确：

但需明确：

1. 这只是本侧占位命名
2. 最终仍需以齐为正式接口文档为准
3. 后续要继续确认：
   - 请求方式
