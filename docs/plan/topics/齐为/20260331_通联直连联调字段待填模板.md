# 2026-03-31 通联直连联调字段待填模板

## 1. 目的

本文档用于把“齐为 / 通联尚未确认的协议字段”与当前代码中的替换点一一对应。

使用方式：

1. 拿到齐为 / 通联正式协议文档后，不要先改代码
2. 先按本文档逐项补齐真实字段、样例、规则
3. 再结合 `docs/plan/20260331_通联直连联调执行清单.md` 确认当前工程状态和执行顺序
4. 按“代码替换点”定位到对应类实现
5. 最后补单测、联调记录和问题清单

本文档只覆盖 **通联 `ProcessServlet` 直连主线**，不覆盖：

1. 旧 `qweimobile` 聚合网关模式
2. 科技平台 SSO / 用户信息接口
3. 云卡完整页面交互主线

## 2. 当前代码主链

当前 `ALLINPAY_DIRECT` 模式的调用链如下：

1. `AllinpayDirectRequestFactory`
2. `AllinpayDirectPayloadMapperRegistry`
3. `AllinpayDirectEnvelopeFactory`
4. `AllinpayDirectProtocolSerializer`
5. `AllinpayDirectRequestPreparer`
6. `AllinpayDirectTransportMapper`
7. `AllinpayDirectHttpExecutor`
8. `AllinpayDirectResponseVerificationStage`
9. `AllinpayDirectResponseParser`

因此后续真实联调时，原则上只需要替换这 5 类协议相关组件：

1. `AllinpayDirectProtocolSerializer`
2. `AllinpayDirectTransportMapper`
3. `AllinpayDirectHttpExecutor`
4. `AllinpayDirectResponseVerificationStage`
5. `AllinpayDirectResponseParser`

## 10. 建议执行顺序

建议后续拿到协议资料后按以下顺序落：

1. 先填第 4 节，确认三个动作的真实接口 / serviceCode
2. 再填第 5 节和第 6 节，明确请求结构、签名和传输口径
3. 再填第 7 节，明确响应签名来源和响应体结构
4. 最后再改 `ResponseParser`，把已验签响应映射到业务 DTO

这样可以避免在响应未明确前反复修改主流程。
