# 2026-04-01 科技平台接口与改造需求

## 文档目的

本文档仅用于和科技平台对齐本次需求中需要提供的能力、接口，以及每个接口仍需确认的信息。

当前业务口径：

1. 借款入口目前在云卡。
2. 云卡调用科技平台完成借款主链。
3. 艾博生调用科技平台完成还款主链。
4. 还款入口链接由艾博生提供，科技平台第三方还款页面点击 `button` 后跳转。
5. 协议列表能力待定。

## 需要科技平台提供的能力清单

### 一、登录 / 登录验证

包括跳转链接、token 校验、获取用户信息接口。


| 能力 / 接口         | 调用方 | 用途                     | 需科技平台确认                          |
| --------------- | --- | ---------------------- | -------------------------------- |
| 登录跳转链接          | 艾博生 | 用户跳转到科技平台完成登录或鉴权       | 跳转 URL、参数要求、回跳规则                 |
| Token 登录态校验接口   | 艾博生 | 艾博生校验当前 token 登录状态是否有效 | 真实 URL、请求方式、token 传递方式、成功/失败判断规则 |
| `/api/users/me` | 艾博生 | 艾博生查询科技平台用户信息          | 真实 URL、token 获取方式、返回字段样例         |

以下接口按当前口径，理解为科技平台回调云卡。


| 能力 / 接口                         | 当前理解                 | 需科技平台确认                    |
| ------------------------------- | -------------------- | -------------------------- |
| `/guide/api/creditStatusNotice` | 科技平台回调云卡，通知授信 / 进件结果 | 回调 URL、字段定义、签名 / 加密口径、重试规则 |
| `/guide/api/loanInfoNotice`     | 科技平台回调云卡，通知放款结果      | 回调 URL、字段定义、签名 / 加密口径、重试规则 |

回调通知还需要统一确认：

1. 真实 URL
2. header 要求
3. 签名算法
4. 加密模式
5. 成功 / 失败响应样例
