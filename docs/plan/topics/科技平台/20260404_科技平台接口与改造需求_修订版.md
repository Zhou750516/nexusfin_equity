# 2026-04-04 科技平台接口与改造需求（修订版）

## 文档目的

本文档以 `docs/plan/20260401_科技平台接口与改造需求.md` 为主线，保留原有业务口径与章节结构，并根据 `docs/third-part/科技平台/科技平台-小花接口文档V1.1.2.docx` 重新修正“能力 / 接口”一列，同时补充每个接口的主要入参、主要返回与仍待确认事项。

## 当前业务口径

1. 借款入口目前在云卡。
2. 云卡调用科技平台完成借款主链。
3. 艾博生调用科技平台完成还款主链。
4. 还款入口链接由艾博生提供，科技平台第三方还款页面点击 `button` 后跳转。
5. 验证码、协议相关接口本次项目不涉及，文档中统一标注为“本次忽略”。

## 修订原则

1. 以 `20260401_科技平台接口与改造需求.md` 的业务需求为主。
2. 以 `科技平台-小花接口文档V1.1.2.docx` 纠正接口归属、接口名称与相对路径。
3. `docx` 中：
   - **机构侧接口定义** = **科技平台提供，云卡 / 艾博生调用**
   - **平台侧接口定义** = **云卡提供，科技平台调用**
4. `docx` V1.1.2 已补充第 2 章机构侧接口地址定义、第 3 章平台侧相对路径；但 `1.6 平台各环境地址` 仍未填测试 / 生产域名，因此当前仅能确认**相对路径**，完整域名仍待科技平台补充。
5. 登录、用户信息、页面跳转类能力在 `docx` 中仍**未定义**，继续保留为“待科技平台补充”。
6. `docx` 中借款试算路径原文为 `/loan/trail`，本次先按文档原值记录，是否为 `trial` 拼写需科技平台确认。
7. 验证码、协议相关接口虽然在最新 `docx` 中有定义，但根据本次项目范围，统一保留记录并标注为**本次忽略**。
