# 2026-04-18 分发页联调检查清单

## 1. 适用范围

本文用于联调分发页当前已落地的最小闭环，验证范围包括：

1. `/api/benefit-dispatch/context/{benefitOrderNo}`
2. `/api/benefit-dispatch/resolve`
3. H5 `joint-dispatch.logic.ts`
4. 联合登录后 `/joint-dispatch` 对 `benefitOrderNo` 的承接

---

## 2. 联调前检查

### 后端

1. 确认后端已启动
2. 确认 `/api/benefit-dispatch/context/{benefitOrderNo}` 可访问
3. 确认 `/api/benefit-dispatch/resolve` 可访问
4. 确认需要登录态时可带 `NEXUSFIN_AUTH` Cookie 访问

### H5

1. 确认 H5 本地开发环境已启动
2. 确认 `/api/*` 仍代理到本地 Java 后端
3. 确认 `/joint-dispatch` 页面可正常打开

---

## 3. 手工执行步骤

### 场景 A：查询分发页上下文

1. 请求：

```text
GET /api/benefit-dispatch/context/{benefitOrderNo}
```

2. 检查返回字段：
   - `benefitOrderNo`
   - `scene`
   - `orderStatus`
   - `allowRedirect`
   - `redirectMode`

### 场景 B：解析分发页跳转目标

1. 请求：

```text
POST /api/benefit-dispatch/resolve
```

2. 请求体：

```json
{
  "benefitOrderNo": "BEN-20260418-001"
}
```

3. 检查返回字段：
   - `allowRedirect`
   - `redirectMode`
   - `supplierUrl`

### 场景 C：H5 分发页状态机

1. 当后端返回：
   - `allowRedirect=true`
   - `redirectMode=DIRECT`
   - `supplierUrl` 非空
2. 检查前端逻辑是否进入：

```text
redirect
```

3. 当后端不满足直接跳转条件时，检查前端逻辑是否进入：

```text
info
```

---

## 4. 关键检查项

1. `benefitOrderNo` 是否在前后端链路中保持一致
2. `allowRedirect` 与 `redirectMode` 是否被 H5 正确识别
3. `supplierUrl` 是否只在允许直跳时返回
4. 分发页日志是否带：
   - `traceId`
   - `bizOrderNo`
