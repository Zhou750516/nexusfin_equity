## 2. 可直接发送版本

辛苦小花侧同学帮忙梳理一份联合登录调用文档，方便艾博生这边接入和联调。

目前我们理解双方已确认的链路是：

> 科技平台携带 token 跳转到艾博生页面 → 艾博生页面请求艾博生后端 → 艾博生后端调用小花后端完成用户身份校验 / 换取。

为避免后续开发和联调反复确认，烦请补充以下内容：

1. **跳转入参**
   - 科技平台跳转到艾博生页面时会携带哪些参数？
   - `token` 的参数名是什么？
   - 是否还会携带订单号、场景标识、渠道标识、时间戳、签名等？
   - 是否可以提供完整跳转 URL 示例？

2. **token 口径**
   - token 有效期是多少？
   - 是否一次性使用？
   - 是否需要签名校验 / 解密？
   - token 失效或校验失败时，错误码和错误文案是什么？

3. **艾博生后端调用小花后端接口**
   - 请求地址
   - 请求方式
   - 请求头
   - 请求体
   - 返回体
   - 是否需要签名、时间戳、防重放、白名单

4. **用户身份返回口径**
   - 联合登录成功后，小花返回的用户唯一标识字段是什么？
   - 该标识是否稳定不变？
   - 后续业务接口是否统一使用该标识？
   - 是否返回手机号、实名状态、渠道信息等基础字段？

5. **失败场景与联调支持**
   - token 无效、用户状态异常、接口调用失败分别如何返回？
   - 是否有测试环境地址？
   - 是否可提供 mock 用户 / mock token / 样例参数？
   - 是否有推荐联调顺序或注意事项？

我们这边拿到文档后，会继续补艾博生侧页面分发、后端登录态建立、异常页处理和联调计划。
