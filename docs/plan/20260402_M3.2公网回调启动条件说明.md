# 2026-04-02 `M3.2` 公网回调启动条件说明

## 1. 文档目的

本文档用于承接 `2026-04-02` 当天 P1 任务，固定当前项目进入 `M3.2 入站回调联调` 前必须具备的公网条件、当前已具备能力、外部待确认项以及阻塞影响。

## 2. 当前本地已具备能力

截至今天，项目在“回调接收能力”上已经具备以下基础：

### 2.1 回调入口已落地

当前已存在以下回调入口：

1. `/api/callbacks/grant/forward`
2. `/api/callbacks/repayment/forward`
3. `/api/callbacks/exercise-equity`
4. `/api/callbacks/refund`

历史联调材料中还约定了以下候选路径：

5. `/api/callbacks/first-deduction`
6. `/api/callbacks/fallback-deduction`

说明：

1. 当前 `NotificationCallbackController` 已覆盖放款、还款、行权、退款
2. 扣款类路径仍需结合最终联调口径确认

### 2.2 回调签名校验已具备

当前 `/api/callbacks/**` 已由签名拦截器保护，要求请求头包含：

1. `X-App-Id`
2. `X-Timestamp`
3. `X-Nonce`
4. `X-Signature`

当前代码口径：

1. 保护路径前缀：`/api/callbacks`
2. 时间戳容忍窗口：`300` 秒
3. 签名算法：`appId + timestamp + nonce + secret` 的本地 HMAC 口径

需要注意：

> 当前签名口径是我方现阶段基线实现，正式联调前仍需确认是否与外部合作方口径一致。

### 2.3 回调业务处理主链已具备

当前本地在收到回调后，已经具备：

1. 参数校验
2. 签名校验
3. 通知落日志
4. 订单状态更新
5. 幂等处理

因此当前 `M3.2` 的主要阻塞已不在本地回调代码主链，而在公网暴露条件和外部约定。

## 3. 当前进入 `M3.2` 前必须确认的外部条件

今天建议固定为以下三项：

### 3.1 公网测试回调域名

必须确认：

1. 对外可访问的测试域名
2. 是否由网关统一转发
3. 是否已有现成测试环境域名可直接复用

如果没有这个域名，齐为无法从外网回调我方测试环境。

### 3.2 回调路径暴露方式

必须确认：

1. 是直接暴露应用路径，还是经网关统一转发
2. 回调路径是否保持当前 `/api/callbacks/**` 结构
3. 是否需要额外 path rewrite
4. 是否需要 HTTPS 终止或特殊代理配置

如果路径暴露方式不明确，即使有公网域名，也无法保证齐为能回调到正确入口。

### 3.3 白名单策略

必须确认：

1. 是否需要来源 IP 白名单
2. 是否需要来源域名白名单
3. 白名单由谁配置
4. 若需要白名单，齐为提供哪些来源信息

如果白名单策略不明确，后续很容易出现：

1. 域名已开通但回调被拦截
2. 环境看似可达但实际联调失败

## 4. 当前固定的阻塞判断

截至 `2026-04-02`，当前应统一使用以下判断：

1. `M3.2` **还不能排期启动**
2. 原因不是本地缺少回调入口
3. 而是以下三项外部条件未确认：
   - 公网测试回调域名
   - 回调路径暴露方式
   - 白名单策略

一句话概括：

> 当前 `M3.2` 的阻塞核心是“外部网络暴露条件未定”，不是“本地回调代码没准备好”。

## 5. 建议对网关 / 运维同步的最小问题清单

后续向我方网关 / 运维同步时，建议只问这三类问题：

1. 是否已有可供第三方访问的测试域名？
2. `/api/callbacks/**` 是否可以直接对外暴露，还是需要网关转发？
3. 若第三方回调需要放开访问，是否要做 IP / 域名白名单？

## 6. 当前推荐同步口径

可直接使用如下口径：

> 当前本地回调入口和签名校验主链已经具备，`M3.2` 暂未启动的原因主要是公网暴露条件未确认。  
> 请协助确认：  
> 1. 可供第三方访问的测试域名  
> 2. `/api/callbacks/**` 的对外暴露方式  
> 3. 是否需要来源 IP / 域名白名单，以及具体配置方式

## 7. 后续收到结果后的处理动作

### 情况 A：三项都确认

则后续可以继续做：

1. 生成完整回调 URL 清单
2. 对齐给第三方
3. 启动 `M3.2` 首轮真实回调联调

### 情况 B：只确认了部分条件

则后续继续做：

1. 把未确认项固定为阻塞说明
2. 不启动 `M3.2`
3. 继续等待剩余外部条件

## 8. 阶段性结论

> 今天公网回调这条线的目标，不是直接启动联调，而是把“为什么还不能启动”说清楚，并固定成后续可直接复用的同步口径。
