# 2026-04-13 H5 多语言支持实施计划

## 1. 目标

基于 `20260413_H5多语言支持方案设计.md`，完成 H5 多语言能力的第一阶段落地，使页面具备：

1. 浏览器语言自动识别
2. 用户手动切换
3. 用户选择持久化
4. 简体中文、繁体中文、英文、越南文四语静态文案展示
5. 后端接口与协议 / 富文本内容的语言治理基础

## 2. 执行顺序

### 2.1 第一阶段：H5 i18n 基础能力

1. 新增语言类型定义与语言归一化工具
2. 新增四语文案资源文件
3. 新增 `I18nProvider`
4. 新增 `useI18n()` 与 `t(key)` 访问能力
5. 新增语言切换组件
6. 在 `App.tsx` 注册全局 Provider

### 2.2 第二阶段：H5 页面文案替换

优先替换当前 6 个 H5 主页面的静态文案：

1. `CalculatorPage`
2. `ApprovalPendingPage`
3. `BenefitsCardPage`
4. `ApprovalResultPage`
5. `ConfirmRepaymentPage`
6. `RepaymentSuccessPage`

替换策略：

1. 保持页面布局和 Tailwind 样式不变
2. 静态短文案改为 `t(key)`
3. 页面数组类文案改为从 `messages` 中读取
4. 金额、日期、银行尾号等业务数据暂不做翻译，只保留格式化空间

### 2.3 第三阶段：后端语言资源基础

1. 新增后端 `LocaleResolver` / `LocaleContext`
2. 后端按 `Accept-Language` 解析语言
3. 新增协议标题与长文案资源结构
4. 让后端 H5 响应中的展示字段具备语言化扩展点

### 2.4 第四阶段：验证

1. H5 执行 TypeScript 类型检查
2. 后端执行 checkstyle
3. 保持 Phase 9 A/C/D/E 集成测试通过
4. 记录尚未接 API 的页面静态数据风险

## 3. 本轮建议边界

本轮优先完成：

1. H5 多语言基础设施
2. H5 页面静态文案切换
3. 后端语言解析与资源骨架
4. 文档更新

本轮暂不强行完成：

1. 数据库化文案中心
2. 运营后台
3. 所有后端业务字段逐项翻译
4. 协议法务正式译文校对

## 4. 验收标准

1. 首次进入页面可根据浏览器语言匹配 `zh-CN`、`zh-TW`、`en-US`、`vi-VN`
2. 用户可手动切换语言
3. 手动切换后刷新页面仍保持用户选择
4. H5 主要页面无核心中文硬编码按钮 / 标题残留
5. 后端具备统一语言解析入口
6. H5 类型检查通过
7. 后端 checkstyle 与 Phase 9 集成测试通过

## 5. 当前执行口径

> 先落前端 i18n 基础和页面静态文案切换，再补后端语言资源骨架，避免在协议内容尚未法务确认前过度建设内容平台。
