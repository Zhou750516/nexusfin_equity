# 惠聚项目开发到上线整体计划

文档日期：2026-03-29
最近更新：2026-04-01

## 1. 计划目标

本文档用于从当前项目状态出发，整理 `nexusfin-equity` 从“当前开发阶段”推进到“联调完成、验收完成、上线发布”的整体路线图。

本计划重点关注两类阶段：

1. 开发侧阶段收口
2. 联调与上线侧阶段推进

目标不是把所有事情铺得很散，而是把真正决定上线节奏的关键里程碑拆清楚。

## 2. 当前项目位置判断

基于截至 2026-03-31 的周报、验收记录、bug 说明、任务回填、联调准备文档和最新验证结果，当前项目大致处于：

**M1 已完成，M2 内部准备已基本完成；截至 2026-04-01，仓库内仍未见关键外部参数、真实测试地址和回调映射规则已确认的新增记录，当前仍卡在外部确认项，等待进入 M3.1 出站接口真实联调。**

当前已具备的基础：

1. 核心基线能力已落地：
   - 订单创建
   - 回调处理
   - 幂等
   - 审计
   - 敏感信息加密
2. 跨域认证主链已落地：
   - SSO 回调登录
   - 本地 JWT + Cookie
   - `/api/users/me`
   - 业务接口认证边界
3. 齐为协议适配层和 3 个主动调用接口已进入代码主链。
4. 当前代码已通过：
   - `mvn test`
   - MySQL 回归
   - `package`
   - `checkstyle`
5. 联调准备文档已形成：
   - 齐为联调前检查清单
   - 待外部确认项清单
   - 出站接口联调执行说明

当前仍未彻底完成的关键事项：

1. 齐为真实 `partnerNo / signKey / aesKeyBase64` 尚未确认，出站接口还停留在 `MOCK` 验证层面。
2. 科技平台真实 `/api/users/me` 地址与 token 样例尚未确认，SSO 真联调无法闭环。

4. 我方公网测试回调域名与白名单尚未确认，`M3.2` 无法启动。
5. 生产化交付文档仍未全部完成，例如上线说明与最终验收结论。
6. 云卡完整页面交互与借还款编排仍未完成，该部分属于独立功能交付主线，不应与齐为联调主线混为同一里程碑判断。

截至 2026-04-01 的补充判断：

1. 今天的主工作不应再表述为“继续补本地开发”，而应表述为“拿确认、推联调、收阻塞”。
2. 若当天拿不到齐为真实参数、科技平台真实地址与 token 样例，`M2` 仍不能从“内部准备完成”推进到“外部条件确认完成”。
3. 若当天拿到关键外部条件，应优先直接启动 `M3.1`，而不是继续新增本地功能性开发。

## 5. 关键依赖关系

项目推进建议按以下依赖顺序执行：

1. **M1 开发完成** 先于 **M2 联调准备完成**
2. **M2 联调准备完成** 先于 **M3 联调完成**
3. **M3 联调完成** 先于 **M4 验收完成**
4. **M4 验收完成** 先于 **M5 上线准备完成**
5. **M5 上线准备完成** 先于 **M6 正式上线**

1. 没有完成联调，不建议直接进入上线验收。

3. 没有准备好回滚和观察项，不建议安排正式发布。

## 11. 推荐判断口径

为了避免项目状态表述失真，后续建议统一用下面的口径描述阶段：

1. **开发完成**：代码主链落地，测试和基础验证通过。
2. **联调完成**：对外关键链路在真实协作环境中打通。
3. **验收完成**：业务、测试、文档、遗留项都已形成明确结论。
4. **上线准备完成**：发布条件、回滚条件、观察项全部就绪。
5. **正式上线**：生产发布完成并经过观察期确认稳定。
