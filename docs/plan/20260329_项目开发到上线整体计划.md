# 惠聚项目开发到上线整体计划

文档日期：2026-03-29  
最近更新：2026-03-31

## 1. 文档目的

本文档用于从当前项目状态出发，梳理 `nexusfin-equity` 从“开发阶段”推进到“联调完成、验收完成、上线发布”的整体路径。

本文档聚焦两件事：

1. 明确齐为联调与上线交付主线的阶段目标、阻塞项和时间线。
2. 明确云卡完整交互属于独立开发主线，避免与当前交付判断混写。

## 2. 当前状态判断

截至 `2026-03-31`，项目整体处于：

## 5. 当前重点建议

截至 `2026-03-31`，近期最关键的工作不是继续横向扩功能，而是推动以下三件事：

| 优先级 | 事项 | 目标 |
| --- | --- | --- |
| 1 | 完成 `M2` 外部确认项 | 拿到齐为真实参数、科技平台真实测试地址与 token 样例，确认回调映射规则和公网回调域名 |
| 2 | 启动 `M3` 真实联调 | 先打通出站接口，再打通回调，最后补异常场景验证 |
| 3 | 提前准备 `M4` 验收材料 | 联调记录、验收记录、上线说明草稿提前形成 |

## 6. 风险与待确认事项

### 6.1 风险提示

| 风险 | 影响 |
| --- | --- |
| 第三方测试环境、密钥、白名单未及时准备 | 直接影响联调节奏 |

## 9. 推荐汇报口径

后续建议统一使用以下阶段口径，避免项目状态表述失真：

| 阶段 | 建议定义 |
| --- | --- |
| 开发完成 | 代码主链落地，测试和基础验证通过 |
| 联调完成 | 对外关键链路在真实协作环境中打通 |
| 验收完成 | 业务、测试、文档、遗留项均形成明确结论 |
| 上线准备完成 | 发布条件、回滚条件、观察项全部就绪 |
| 正式上线 | 生产发布完成并经过观察期确认稳定 |

补充建议：

1. 后续汇报中，始终区分“齐为联调进度”和“云卡完整交互开发进度”。
2. 不再使用单一“联调/交付进度”同时覆盖两条主线，避免状态失真。
