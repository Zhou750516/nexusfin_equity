1. `plan/20260420_项目阶段性TODO清单.md`
2. `plan/20260420_项目阶段性TODO_checklist.md`

1. 想快速知道齐为、小花科技、电子签章当前还有哪些未完成事项；

3. 想知道哪些事项已经收口、哪些仍不能抢跑开发。

2. 了解当天边界；
3. 跟踪当天完成情况和剩余事项。

## 2. 后续维护规则

为避免文档继续散落，建议后续统一按下面规则维护：

### 2.1 外部反馈先更新 checklist

若外部有新反馈，优先更新：

1. `plan/20260420_项目阶段性TODO_checklist.md`

适合更新的内容：

1. 文档是否已收到；
2. 字段是否已确认；
3. 哪些仍待外部答复；
4. 某个阶段是否可以从 `[ ]` 改为 `[~]` 或 `[x]`。

### 2.2 方案收口后回填专题文档

当某事项已进入本侧方案阶段时：

1. 新建或更新对应专题方案文档；
2. 在阶段性 TODO 清单里回填状态；
3. 在当日 plan / 日报 / decision-review 中引用该专题文档。

### 2.3 开发启动后拆独立实施计划

当某事项进入真实开发或联调阶段时：

1. 单独补实施计划；
2. 单独补开发任务拆解；
3. 单独补测试与验证文档；
4. 不再只停留在阶段性 TODO 清单里。
