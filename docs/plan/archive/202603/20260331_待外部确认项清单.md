## 3. 建议对外同步口径

### 3.1 发给齐为

请协助确认以下信息：

1. 测试环境最终网关地址、`methodPath` 和协议版本口径。
2. 联调所需 `partnerNo`、`signKey`、`aesKeyBase64`。
3. `abs.push.order`、`abs.token.get`、`abs.lending.notify` 三个接口的成功 / 失败样例。
4. 扣款通知字段定义，以及如何区分首扣与兜底代扣。
5. 行权、退款、还款回调的最小样例报文。

### 3.2 发给科技平台

请协助确认以下信息：

1. 测试环境 `baseUrl` 和 `/api/users/me` 的真实地址。
2. Bearer Token 获取方式或可直接使用的测试 token 样例。
3. 返回字段口径是否固定包含：
   - `userId`
   - `phone`
   - `realName`
   - `idCard`
   - `channelCode`
4. 若字段名或结构与当前实现不同，请提供真实响应样例。

### 3.3 发给我方网关 / 运维

请协助确认以下信息：

1. 可供齐为访问的测试域名或网关地址。
2. 对应回调完整路径是否可直接暴露：
   - `/api/callbacks/first-deduction`
   - `/api/callbacks/fallback-deduction`
   - `/api/callbacks/grant/forward`
   - `/api/callbacks/repayment/forward`
   - `/api/callbacks/exercise-equity`
   - `/api/callbacks/refund`
3. 是否需要齐为侧提前做来源 IP 或域名白名单配置。

## 4. 当前阻塞说明

当前最关键的阻塞不在本地代码，而在外部协同项：

1. 没有齐为真实参数，出站接口只能停留在 `MOCK` 验证。
2. 没有科技平台真实环境，SSO 真实联调无法闭环。
3. 没有公网回调地址，入站回调联调无法开始。
4. 没有扣款回调映射规则，`first-deduction / fallback-deduction` 无法稳定路由。
