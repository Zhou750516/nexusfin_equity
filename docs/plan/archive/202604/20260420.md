1. 若过早把自动调度默认改为开启，容易污染现有联调与测试环境
2. 若在 `currentTaskId` 等细节未想清楚前贸然扩写，后续可能返工
3. 若把调度层写入业务逻辑，后续会破坏当前清晰的 service 分层
4. 若扩面到退款 / 绑卡 / 状态回推补偿，容易打散当前主线
