## 0. 当前环境判断

### 0.1 结论

这份清单**有必要保留**，但它不是生产上线手册，而是 2026-05-08 阿里云单机真实联调的最小执行入口。

当前最小目标是：

1. 先把 `Yunka / 小花科技` 真实测试环境打通。
2. `QW` 继续保持 `MOCK`。
3. 科技平台继续使用本机 `tech-user-stub`。
4. H5 通过 Node production server 跑在 `3000`，由 nginx 挂载到 `https://www.aibosoftware.com/equity/`。
5. nginx 保留官网首页 `/` 静态站，`/api/` 转发后端，`/equity/` 转发 H5。

### 0.2 必要项

这些项和当前代码、脚本、联调目标一致，必须保留：

1. `Java 17`：后端是 Spring Boot 3.2.12。
2. `Maven`：`scripts/aliyun/start-backend.sh` 在缺少 jar 或强制重建时会执行 `mvn -q -DskipTests package`。
3. `Node.js 20+` 和 `pnpm`：H5 生产模式是 `pnpm build` 后 `pnpm start`。
4. `nginx`：对外暴露 `80/443`，将 `/api/` 转发到 `backend:8080`，将 `/equity/` 转发到 `H5:3000`，并保留官网首页 `/`。
5. `git`：用于拉取和更新当前仓库。
6. `MySQL 8.0`：后端必须连接 MySQL。本次按 ECS 本机 MySQL 执行，先检查，未安装再装。
7. `tech-user-stub`：当前口径下科技平台不接真实服务，后端指向 `http://127.0.0.1:18080`。
8. `YUNKA_MODE=REST`：今天真实联调主线是 Yunka，不能继续跑默认 `MOCK`。
9. `QW_MODE=MOCK`：第一天不建议和 Yunka 同时切真。

### 0.3 可选项

这些项按现场条件选择，不是第一天联调的硬前置：

1. `HTTPS`：本次域名明确为 `https://www.aibosoftware.com/`，必须启用；`AUTH_COOKIE_SECURE=true`。
2. `systemd`：当前仓库没有 systemd 服务文件，先用现有脚本和 pid 文件联调；稳定后再补守护进程。

### 0.4 暂不做项

这些项当前没有必要放进执行路径：

1. 不上 Docker：仓库没有 Dockerfile / docker-compose，今天引入会扩大部署变量。
2. 不把 `8080 / 3000 / 18080 / 3306` 对公网开放：只允许本机回环或内网访问。
3. 不接 RDS：本次 MySQL 固定为本机安装。
4. 不默认接真实科技平台：除非云卡明确存在必须直连边界。
5. 不默认切真实 QW：`benefiturl` 运行时来源仍是 QW exercise redirect URL，第一天同时切真会放大定位面。

```bash

```

如果你的 MySQL 已经完成初始化，可以直接继续下一步。

## 7. 第六步：一键启动联调环境

在代码目录执行：

```bash
cd ~/project/nexusfin_equity
source .env.aliyun
./scripts/aliyun/start-all.sh
```

这条命令会顺序启动：

1. `tech-user-stub`，端口 `18080`
2. backend，端口 `8080`
3. H5，端口 `3000`

脚本行为说明：

1. backend 如果找不到 jar，会自动执行 `mvn -q -DskipTests package`。
2. H5 如果没有依赖，会自动执行 `pnpm install`。
3. H5 如果没有生产构建，会自动执行 `pnpm build`。
4. H5 生产服务通过 `pnpm start` 启动，监听 `3000`。

运行日志默认在：

```text
~/project/nexusfin_equity/runtime/aliyun/
```

查看日志：

```bash
tail -f ~/project/nexusfin_equity/runtime/aliyun/tech-user-stub.log
tail -f ~/project/nexusfin_equity/runtime/aliyun/backend.log
tail -f ~/project/nexusfin_equity/runtime/aliyun/h5.log
```

如果要停止：

```bash
cd ~/project/nexusfin_equity
./scripts/aliyun/stop-all.sh
```

如果要强制重新打包后端和 H5：

```bash
cd ~/project/nexusfin_equity
source .env.aliyun
FORCE_REBUILD=1 ./scripts/aliyun/start-all.sh
```

## 10. 第九步：配置 nginx

### 10.1 推荐：执行脚本自动应用

当前仓库已提供 nginx 配置应用脚本：

```bash
cd ~/project/nexusfin_equity
./scripts/aliyun/apply-nginx-aibosoftware.sh
```

脚本会自动执行：

1. 备份 `/etc/nginx/nginx.conf`
2. 写入 `www.aibosoftware.com` 的 nginx 配置
3. 执行 `sudo nginx -t`
4. 检查通过后执行 `sudo systemctl reload nginx`
5. 如果 `nginx -t` 失败，会回滚到备份配置

### 10.2 手工复制样板

复制样板配置：

```bash
cp ~/project/nexusfin_equity/deploy/nginx/nexusfin-equity-single-server.conf /etc/nginx/conf.d/nexusfin-equity.conf
```

把配置里的：

```nginx
server_name _;
```

改成当前域名：

```nginx
server_name www.aibosoftware.com;
```

检查配置：

```bash
nginx -t
```

启动并设置开机自启：

```bash
systemctl enable nginx
systemctl restart nginx
systemctl status nginx --no-pager
```

nginx 默认会把：

1. `/api/` 转发到 `127.0.0.1:8080`
2. `/equity/` 转发到 `127.0.0.1:3000`
3. `/` 继续保留官网静态首页 `/usr/share/nginx/html`

当前按 `https://www.aibosoftware.com/` 联调，必须启用 443 server block，并保留：

1. `/api/` -> `127.0.0.1:8080`
2. `/equity/` -> `127.0.0.1:3000`
3. `/equity` -> `301 /equity/`
4. `X-Forwarded-Proto https`

### 13.7 判定环境没配对

看到下面任一现象，先按环境问题处理：

1. backend 日志里 `mode=MOCK`，说明 `YUNKA_MODE` 没生效。
2. backend 日志里 `baseUrl=https://yunka.local`，说明 `YUNKA_BASE_URL` 没生效。
3. backend 启动脚本提示缺少 `YUNKA_BASE_URL / YUNKA_CHANNEL_CODE / YUNKA_SIGNATURE`，说明 `.env.aliyun` 没 source 或变量没填。
4. 通过 `https://www.aibosoftware.com/` 登录后 cookie 不写入，检查 `AUTH_COOKIE_SECURE=true` 和 443 nginx 配置。
5. nginx 返回 502，先查 `ss -lntp` 确认 `8080 / 3000` 是否监听。
6. `/api/users/me` 401，先确认 `tech-user-stub` 是否启动，且 token 是 `mock-tech-token`。
