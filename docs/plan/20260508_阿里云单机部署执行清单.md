# 2026-05-08 阿里云单机部署执行清单

## 1. 适用范围

这份清单只针对当前最小联调方案：

1. 一台阿里云服务器
2. 先联调 Yunka
3. 不上 Docker
4. QW 先保持 `MOCK`
5. 科技平台不接真实服务，先用本地 `tech-user-stub`

默认操作系统按 `Alibaba Cloud Linux 3` 写。

---

## 2. 第一步：安装基础软件

先切到 `root`：

```bash
sudo -i
```

更新系统：

```bash
dnf makecache
dnf update -y
```

安装 Git、Java、Maven、Nginx、MySQL：

```bash
dnf install -y git java-17-openjdk java-17-openjdk-devel maven nginx mysql mysql-server
```

安装 Node.js 20：

```bash
curl -fsSL https://rpm.nodesource.com/setup_20.x | bash -
dnf install -y nodejs
```

安装 pnpm：

```bash
npm install -g pnpm
```

检查版本：

```bash
java -version
mvn -version
node -v
pnpm -v
nginx -v
mysql --version
```

---

## 3. 第二步：启动 MySQL

启动并设置开机自启：

```bash
systemctl enable mysqld
systemctl start mysqld
systemctl status mysqld --no-pager
```

如果是首次安装，先看临时 root 密码：

```bash
grep 'temporary password' /var/log/mysqld.log || true
```

如果你的 MySQL 已经完成初始化，可以直接继续下一步。

---

## 4. 第三步：拉代码

先切到你准备部署的目录，例如 `/opt`：

```bash
cd /opt
git clone <你的仓库地址> nexusfin-equity
cd /opt/nexusfin-equity
```

如果不是首次部署，直接更新代码：

```bash
cd /opt/nexusfin-equity
git pull
```

---

## 5. 第四步：创建数据库、work 账号、导入表

执行我们刚补好的脚本：

```bash
cd /opt/nexusfin-equity
export MYSQL_ADMIN_PASSWORD='你的MySQL管理员密码'
export WORK_DB_PASSWORD='你要给work设置的密码'
./scripts/setup-aliyun-mysql.sh
```

如果你要手工确认，可以再跑：

```bash
mysql -uroot -p -D nexusfin_equity -e "show tables;"
mysql -uroot -p -e "show grants for 'work'@'localhost';"
mysql -uroot -p -e "show grants for 'work'@'127.0.0.1';"
```

---

## 6. 第五步：准备运行环境变量

先在服务器上建一个本地环境文件：

```bash
cat >/opt/nexusfin-equity/.env.aliyun <<'EOF'
export MYSQL_HOST='127.0.0.1'
export MYSQL_PORT='3306'
export MYSQL_DATABASE='nexusfin_equity'
export MYSQL_USERNAME='work'
export MYSQL_PASSWORD='替换成work密码'

export AUTH_JWT_SECRET='替换成你自己的jwt密钥'
export NEXUSFIN_APP_SECRET='替换成你自己的app密钥'

export YUNKA_BASE_URL='替换成云卡真实测试地址'
export YUNKA_CHANNEL_CODE='替换成云卡确认值'
export YUNKA_SIGNATURE='替换成云卡确认值'

export QW_MODE='MOCK'

# 如果当前只走 http，先用 false
# 如果 nginx 已经配好 https，再改成 true
export AUTH_COOKIE_SECURE='false'
EOF
```

生效：

```bash
cd /opt/nexusfin-equity
source .env.aliyun
```

检查关键变量：

```bash
env | grep -E 'MYSQL_|YUNKA_|QW_MODE|AUTH_COOKIE_SECURE|AUTH_JWT_SECRET|NEXUSFIN_APP_SECRET'
```

---

## 7. 第六步：一键启动联调环境

在代码目录执行：

```bash
cd /opt/nexusfin-equity
source .env.aliyun
./scripts/aliyun/start-all.sh
```

这条命令会顺序启动：

1. `tech-user-stub`，端口 `18080`
2. backend，端口 `8080`
3. H5，端口 `3000`

运行日志默认在：

```text
/opt/nexusfin-equity/runtime/aliyun/
```

查看日志：

```bash
tail -f /opt/nexusfin-equity/runtime/aliyun/tech-user-stub.log
tail -f /opt/nexusfin-equity/runtime/aliyun/backend.log
tail -f /opt/nexusfin-equity/runtime/aliyun/h5.log
```

如果要停止：

```bash
cd /opt/nexusfin-equity
./scripts/aliyun/stop-all.sh
```

---

## 8. 第七步：检查端口

先确认 3 个进程都真的起来了：

```bash
ss -lntp | grep -E ':18080|:8080|:3000'
```

期望看到：

1. `18080`：tech-user-stub
2. `8080`：backend
3. `3000`：H5

---

## 9. 第八步：检查后端启动真值

先看 backend 日志里的关键字：

```bash
grep -E "yunka gateway client initialized|qw benefit client routing initialized|Started" /opt/nexusfin-equity/runtime/aliyun/backend.log
```

你要重点确认：

1. `Yunka mode=REST`
2. `Yunka baseUrl=你配置的真实地址`
3. `QW mode=MOCK`

如果这里不对，先别往下跑页面。

---

## 10. 第九步：配置 nginx

复制样板配置：

```bash
cp /opt/nexusfin-equity/deploy/nginx/nexusfin-equity-single-server.conf /etc/nginx/conf.d/nexusfin-equity.conf
```

检查配置：

```bash
nginx -t
```

启动并设置开机自启：

```bash
systemctl enable nginx
systemctl restart nginx
systemctl status nginx --no-pager
```

nginx 默认会把：

1. `/api/` 转发到 `127.0.0.1:8080`
2. `/` 转发到 `127.0.0.1:3000`

---

## 11. 第十步：放行安全组

阿里云安全组至少放行：

1. `22`
2. `80`
3. `443`，如果你准备上 HTTPS

不要直接对公网放：

1. `3000`
2. `8080`
3. `18080`
4. `3306`

---

## 12. 第十一步：最小 smoke

### 12.1 健康检查

```bash
curl -i http://127.0.0.1:8080/api/equity/health
curl -i http://127.0.0.1/
curl -i http://127.0.0.1/api/equity/health
```

### 12.2 tech-user-stub 自检

```bash
curl -i http://127.0.0.1:18080/api/users/me -H 'Authorization: Bearer mock-tech-token'
```

### 12.3 联调前 Cookie 准备

先打一次登录入口：

```bash
curl -i -c /tmp/nexusfin_auth.txt "http://127.0.0.1:8080/api/auth/sso-callback?token=mock-tech-token"
```

如果后端已经挂在 nginx 后，也可以用：

```bash
curl -i -c /tmp/nexusfin_auth.txt "http://127.0.0.1/api/auth/sso-callback?token=mock-tech-token"
```

### 12.4 最小后端链路

```bash
curl -i http://127.0.0.1:8080/api/equity/health
```

如果你已经要开始真正联调，再继续跑：

1. `joint-login`
2. `loan/calculate`
3. `benefits/card-detail`
4. `benefits/activate`
5. `repayment/info`

---

## 13. 常用排障命令

### 13.1 看 backend 最近 200 行日志

```bash
tail -n 200 /opt/nexusfin-equity/runtime/aliyun/backend.log
```

### 13.2 看 H5 最近 100 行日志

```bash
tail -n 100 /opt/nexusfin-equity/runtime/aliyun/h5.log
```

### 13.3 看 nginx 错误日志

```bash
tail -n 100 /var/log/nginx/error.log
```

### 13.4 重启整套服务

```bash
cd /opt/nexusfin-equity
./scripts/aliyun/stop-all.sh
source .env.aliyun
./scripts/aliyun/start-all.sh
```
