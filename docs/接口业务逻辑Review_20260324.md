# 接口业务逻辑 Review

## 1. `GET /api/auth/sso-callback`

Review 点：

1. 上游 token 校验失败时是否返回清晰错误
2. `redirect_url` 是否严格走白名单
3. JIT 建档是否能复用已有会员且不重复建档
4. JWT Cookie 是否具备 `HttpOnly`、`SameSite`、`Secure` 配置

## 2. `GET /api/users/me`

Review 点：

1. 是否只信任本地 JWT，不接受客户端自带 `memberId`
2. 会员不存在或 token 失效时是否返回未认证

## 3. `GET /api/equity/products/{productCode}`

Review 点：

1. 是否由认证上下文获取当前 `memberId`
2. 是否移除了显式 `memberId` 查询参数依赖

## 4. `POST /api/equity/orders`

Review 点：

1. 是否由认证上下文获取当前 `memberId`
2. 请求体是否已移除 `memberId`
3. 幂等、协议生成、订单写库是否仍保持原有行为

## 5. `/api/callbacks/**`

Review 点：

1. 是否仍保留签名拦截
2. 是否未被 JWT 过滤器误伤
3. 业务状态机是否保持基线一致
