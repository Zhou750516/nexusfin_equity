# 艾博生权益分发服务接口业务处理逻辑

## 1. 当前业务主链

更新后的主链已经从“静默注册 + 下单”切换为“SSO 登录 + 下单”：

1. `GET /api/auth/sso-callback`
2. `GET /api/users/me`
3. `GET /api/equity/products/{productCode}`
4. `POST /api/equity/orders`
5. `POST /api/callbacks/first-deduction`
6. `POST /api/callbacks/grant/forward`
7. `GET /api/equity/orders/{benefitOrderNo}`
8. `GET /api/equity/exercise-url/{benefitOrderNo}`
9. `POST /api/callbacks/exercise-equity`
10. `POST /api/callbacks/refund`
11. `POST /api/callbacks/repayment/forward`

`POST /api/users/register` 已下线。

## 2. 认证链路

### 2.1 `GET /api/auth/sso-callback`

处理逻辑：

1. 接收科技平台透传的 `token`
2. 服务端调用科技平台用户接口校验 token
3. 按 `tech_platform_user_id` 查找本地会员
4. 若本地会员不存在，则即时建档：
   - 写 `member_info`
   - 写 `member_channel`
5. 生成本地 JWT
6. 通过 `Set-Cookie` 回写浏览器
7. `302` 跳转到白名单内目标页面

### 2.2 `GET /api/users/me`

处理逻辑：

1. 读取本地 JWT Cookie
2. 解析出 `memberId` 与 `techPlatformUserId`
3. 查询 `member_info`
4. 返回当前登录用户信息

## 3. 下单链路

### 3.1 `GET /api/equity/products/{productCode}`

处理逻辑：

1. JWT 过滤器先校验本地登录态
2. Controller 从认证上下文获取当前 `memberId`
3. Service 查询产品信息
4. Service 查询当前会员信息用于回显
5. 返回产品页展示数据

变化点：

- 不再接受外部显式传入的 `memberId`

### 3.2 `POST /api/equity/orders`

处理逻辑：

1. JWT 过滤器先校验本地登录态
2. Controller 从认证上下文获取当前 `memberId`
3. Service 进行请求幂等判断
4. 查询产品、会员、渠道绑定
5. 写入 `benefit_order`
6. 生成协议任务与合同归档
7. 写入幂等记录
8. 返回 `benefitOrderNo`

变化点：

- 请求体已移除 `memberId`
- 订单归属完全由登录态决定

## 4. 回调链路

### 4.1 回调认证

以下接口仍由签名拦截器保护，而不是本地 JWT：

- `POST /api/callbacks/first-deduction`
- `POST /api/callbacks/fallback-deduction`
- `POST /api/callbacks/grant/forward`
- `POST /api/callbacks/repayment/forward`
- `POST /api/callbacks/exercise-equity`
- `POST /api/callbacks/refund`

### 4.2 回调处理

支付、放款、还款、行权、退款的业务处理逻辑与原基线保持一致：

1. 幂等校验
2. 回调日志留痕
3. 订单状态机流转
4. 审计与对账数据更新

## 5. 关键实现差异

### 5.1 已移除

- `/api/users/register`
- 显式注册 DTO 与注册 Service

### 5.2 已新增

- `AuthController`
- `AuthService`
- `TechPlatformUserClient`
- `JwtAuthenticationFilter`
- `JwtUtil`
- `CookieUtil`

### 5.3 已调整

- `CreateBenefitOrderRequest` 不再接收 `memberId`
- `/api/equity/**` 业务接口改由本地 JWT 保护
- `/api/callbacks/**` 继续保留签名保护
