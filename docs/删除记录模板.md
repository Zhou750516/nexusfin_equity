# 文档删除记录模板

> 用途：凡是计划删除 `docs/` 下的文件，先复制本模板形成一份具体删除记录，再执行删除操作。

## 1. 基本信息

- 记录日期：

## 2. 待删除文件

请逐项列出待删除文件。

1. `待删除文件路径`
2. `待删除文件路径`

## 3. 删除原因

请明确说明删除原因，至少覆盖以下一种或多种情况：

1. 文档已过期
2. 文档内容错误且已由新文档替代
3. 文档重复，保留主版本即可
4. 文档误建 / 误传 / 无有效内容

## 4. 替代关系

若有替代文档，请写清：

1. 替代文档路径：
2. 替代关系说明：

若没有替代文档，请明确写：

- 无直接替代文档，删除后不再保留该主题独立文档。

## 5. 是否可从 Git 历史恢复

- 是否可恢复：`是 / 否`
- 最近可恢复提交：
- 恢复方式说明：

示例：

```bash
git log -- 文件路径
git restore --source=<commit>^ -- 文件路径
```

## 6. 影响范围评估

请说明删除后可能影响的范围：

1. 是否影响现有计划 / checklist / 日报 / decision-review 的引用
2. 是否影响外部对接、联调、上线说明的引用
3. 是否需要同步更新其他文档中的链接或说明

## 7. 删除前检查

- [ ] 已确认文件确实需要删除
- [ ] 已确认是否存在替代文档
- [ ] 已确认是否可从 Git 历史恢复
- [ ] 已检查是否有其他文档引用该文件
- [ ] 已完成本删除记录
